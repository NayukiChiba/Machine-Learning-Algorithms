# 评估与诊断

> 对应代码：`pipelines/probabilistic/hmm.py`、`model_training/probabilistic/hmm.py`
>  
> 相关对象：`states_pred`、`model.transmat_`

## 本章目标

1. 明确当前仓库实际使用了哪些评估手段，而不是泛泛讨论所有序列模型指标。
2. 理解隐状态预测准确率和转移矩阵分别能帮助我们诊断什么。
3. 明确当前实现没有做哪些更完整的 HMM 评估输出，以免误读源码能力边界。

## 重点方法与概念速览

| 名称 | 类型 | 作用 |
|---|---|---|
| `states_pred` | 预测结果 | 当前 HMM 解码得到的隐状态路径 |
| `y_true` | 对比标签 | 数据生成时记录的真实隐状态 |
| `accuracy = np.mean(states_pred == y_true)` | 派生量 | 当前流水线的路径级粗略对比指标 |
| `model.transmat_` | 属性 | 学习得到的状态转移矩阵 |

## 1. 当前实现真正做了什么评估

### 参数速览（本节）

适用评估手段（本节）：

1. 隐状态预测准确率
2. 转移矩阵打印

| 评估方式 | 来源 | 用途 |
|---|---|---|
| 隐状态预测准确率 | `np.mean(states_pred == y_true)` | 粗略比较解码路径和真实隐状态路径 |
| 转移矩阵打印 | `model.transmat_` | 观察模型学到的状态转移结构 |

### 理解重点

- 当前 HMM 流水线没有显式打印对数似然曲线、困惑度、前向概率或更复杂的序列评估指标。
- 这并不表示这些指标不重要，而是说明本仓库当前实现更强调“隐状态路径是否大致对上”和“转移结构长什么样”。
- 因此阅读这一分册时，不能把更完整的 HMM 诊断面板想成已经在源码里实现的内容。

## 2. 隐状态预测准确率是怎么得到的

### 参数速览（本节）

适用代码：`accuracy = np.mean(states_pred == y_true)`

| 参数名 | 本例取值 | 说明 |
|---|---|---|
| `states_pred` | `model.predict(X_obs, lengths)` 输出 | 预测隐状态路径 |
| `y_true` | `state_true` 数组 | 数据生成时的真实隐状态 |
| `accuracy` | `0` 到 `1` 间浮点数 | 两条状态路径按时间步逐点比较的一致比例 |

### 示例代码

```python
states_pred = model.predict(X_obs, lengths)
accuracy = np.mean(states_pred == y_true)
print(f"\n隐状态预测准确率: {accuracy:.4f}")
```

### 理解重点

- 当前准确率的含义很直接：逐时间步比较预测隐状态和真实隐状态是否一致。
- 这是一个教学上容易理解的指标，但它并不一定覆盖 HMM 评估的全部复杂性。
- 特别要注意，当前数据是人为构造且真实隐状态可见，所以这种对比才成立。

## 3. 看转移矩阵时重点观察什么

### 参数速览（本节）

适用属性：`model.transmat_`

| 观察点 | 可以帮助判断什么 |
|---|---|
| 对角元素是否较大 | 状态是否倾向停留在自身 |
| 非对角元素分布 | 状态之间切换是否频繁 |
| 不同行差异 | 各状态动态是否不同 |

### 示例代码

```python
print(f"转移矩阵:\n{model.transmat_.round(3)}")
```

### 理解重点

- 转移矩阵是当前 HMM 分册里最重要的结构性输出之一。
- 它帮助你看到模型学到的“状态如何随时间演化”的规律，而不仅仅是某个时间点预测对不对。
- 如果你只看准确率，不看转移矩阵，就会错过 HMM 最有特点的一部分结果解释。

## 4. 当前准确率指标的边界

### 理解重点

- 当前准确率是建立在我们手里恰好有 `state_true` 的前提下计算出来的。
- 在真实应用里，隐状态通常本来就是看不见的，因此这种指标往往不可直接获得。
- 也就是说，它更适合当前教学数据，不应被误认为 HMM 通用评估标准。

## 5. 当前实现没有做什么

### 参数速览（本节）

当前源码未包含的内容：

1. 前向概率或序列似然打印
2. 维特比路径概率输出
3. 发射矩阵打印
4. 多序列交叉验证或学习曲线

| 未实现项 | 当前状态 |
|---|---|
| `score(...)` 或对数似然打印 | 未在流水线中出现 |
| Viterbi 路径得分 | 未显式打印 |
| 发射矩阵输出 | 未显式打印 |
| 学习曲线 | 当前流程未调用相关函数 |

### 理解重点

- 评估章节必须以源码为准，不能把“HMM 常见分析手段”写成“当前仓库已经实现”。
- 当前实现的评估重点是路径级准确率和转移矩阵，而不是更全面的概率诊断面板。
- 如果后续扩展这部分，最自然的方向是补序列似然、发射矩阵和多序列评估输出。

## 常见坑

1. 只看准确率，不看转移矩阵，误把 HMM 当成普通逐点分类器。
2. 看到准确率不高就直接否定模型，忽略了状态标签本身和路径结构解释也很重要。
3. 误以为当前流水线已经输出了完整前向概率、Viterbi 得分或发射矩阵，实际源码并没有这一步。

## 小结

- 当前 HMM 的评估主线由两部分组成：隐状态预测准确率和转移矩阵。
- 准确率负责看解码路径是否大致对上，转移矩阵负责看状态演化结构是否合理。
- 只有把这两条线索一起看，才能更完整地理解当前实现的表现。
