# 评估与诊断

> 对应代码：`pipelines/clustering/dbscan.py`、`model_training/clustering/dbscan.py`、`result_visualization/cluster_plot.py`
>
> 相关对象：`model.labels_`、`plot_clusters(...)`

## 本章目标

1. 明确当前仓库 DBSCAN 实现实际上是如何做结果诊断的。
2. 理解簇数量和噪声点数量分别能说明什么。
3. 理解为什么当前分册把可视化对照作为主要评估入口。

## 重点方法与概念速览

| 名称 | 类型 | 作用 |
|---|---|---|
| `model.labels_` | 属性 | 反映每个样本的聚类结果，噪声记为 `-1` |
| `簇数量` | 统计量 | 反映当前分出的有效簇个数 |
| `噪声点数量` | 统计量 | 反映被判为离群或未归类的样本数 |
| `plot_clusters(...)` | 函数 | 绘制预测标签图和真实标签图 |
| `labels_pred=model.labels_` | 参数 | 提供预测簇标签 |
| `labels_true=y_true` | 参数 | 提供真实标签用于对照 |

## 1. 当前仓库的评估入口

当前 DBSCAN 流水线里的主要结果诊断手段有两个：

1. 终端日志里打印 `簇数量` 和 `噪声点数量`
2. 调用 `plot_clusters(...)` 绘制聚类分布图

### 示例代码

```python
labels = model.labels_
n_clusters = len(set(labels)) - (1 if -1 in labels else 0)
n_noise = (labels == -1).sum()

plot_clusters(
    X_scaled,
    labels_pred=model.labels_,
    labels_true=y_true,
    title="DBSCAN 聚类分布",
    dataset_name=DATASET,
    model_name=MODEL,
)
```

### 理解重点

- 当前实现没有引入复杂的聚类指标面板，而是优先提供最直观的数值与图像反馈。
- 对教学型仓库来说，这样的设计非常务实：读者可以先看簇是否按形状分开，再理解噪声点意味着什么。
- 由于数据本身是二维的，图像对照在这里尤其有效。

## 2. 簇数量表示什么

### 参数速览（本节）

适用统计量：`n_clusters`

| 名称 | 当前含义 | 说明 |
|---|---|---|
| `n_clusters` | 排除 `-1` 后的簇数量 | 表示 DBSCAN 最终识别出的有效簇个数 |

### 理解重点

- 簇数量反映当前参数配置下，DBSCAN 认为数据里存在多少片彼此分离的高密度区域。
- 它能帮助读者快速判断当前 `eps` 和 `min_samples` 是否过松或过严。
- 但簇数量本身也不能单独证明聚类就一定合理，还需要结合图像结构一起看。

## 3. 噪声点数量表示什么

### 参数速览（本节）

适用统计量：`n_noise`

| 名称 | 当前含义 | 说明 |
|---|---|---|
| `n_noise` | `labels_ == -1` 的样本数 | 表示被 DBSCAN 判为噪声的点数 |

### 理解重点

- 噪声点并不一定意味着模型失败，它往往表示某些样本没有落在足够密集的区域中。
- 如果噪声点过多，可能说明 `eps` 过小或 `min_samples` 过大。
- 如果几乎没有噪声且簇被过度连通，也可能说明 `eps` 过大。

## 4. 可视化对照图能观察什么

`plot_clusters(...)` 会生成一张包含两幅子图的图片：

- 左图：预测簇标签
- 右图：真实标签

### 示例代码

```python
plot_clusters(
    X_scaled,
    labels_pred=model.labels_,
    labels_true=y_true,
)
```

### 理解重点

- 这种对照图最适合回答一个直观问题：模型分出来的簇，是否和数据生成时的真实结构大致一致。
- 观察重点不在“颜色编号是否一一对应”，而在簇的形状、连通性和噪声点位置是否合理。
- 对 DBSCAN 来说，`-1` 这一类尤其值得观察，因为它直接对应噪声样本。

## 5. 当前实现中尚未纳入的量化指标

在一般聚类任务中，还常见以下指标：

- 轮廓系数（Silhouette Score）
- 调整兰德指数（ARI）
- 归一化互信息（NMI）

### 理解重点

- 当前仓库并没有在 DBSCAN 流水线中实现这些指标。
- 文档可以提到它们是常见扩展方向，但不能写成“当前源码已经在计算”。
- 现阶段最准确的表述是：当前实现以簇数量、噪声点数量和二维可视化为主要诊断手段。

## 常见坑

1. 只看簇数量，不看图像分布。
2. 看到 `-1` 就误判模型失败。
3. 看到预测标签编号和真实标签编号不一致，就误判模型失败。
4. 把当前仓库未实现的 ARI、NMI、轮廓系数写成现有流程的一部分。

## 小结

- 当前仓库对 DBSCAN 的评估方式很明确：数值上看簇数量与噪声点数量，图像上看聚类分布对照图。
- `-1` 是 DBSCAN 评估中最需要重点理解的特殊标签。
- 对当前二维教学数据而言，可视化是最直观、也最符合本仓库目标的诊断方式。
