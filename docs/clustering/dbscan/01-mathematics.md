# 数学原理

> 对应代码：`model_training/clustering/dbscan.py`、`pipelines/clustering/dbscan.py`
>
> 对应对象：`DBSCAN(...)`、`model.fit(X_train)`、`model.labels_`

## 本章目标

1. 理解 DBSCAN 是如何用密度而不是中心来定义簇的。
2. 理解 `eps` 和 `min_samples` 如何共同决定核心点、边界点和噪声点。
3. 理解为什么 DBSCAN 能处理任意形状簇，但对密度差异较大的数据会更敏感。

## 重点方法与概念速览

| 名称 | 类型 | 作用 |
|---|---|---|
| `eps` 邻域 | 概念 | 定义每个点周围“多近算邻居” |
| 核心点 | 点类型 | 邻域内样本数达到阈值的点 |
| 边界点 | 点类型 | 自身不是核心点，但落在核心点邻域内 |
| 噪声点 | 点类型 | 不属于任何簇的离散点 |
| 密度直达 | 关系 | 从核心点向邻域内点的一步可达关系 |
| 密度可达 | 关系 | 通过若干步密度直达扩展出的连通关系 |
| 密度相连 | 关系 | 由同一密度连通结构连接的点集 |

## 1. DBSCAN 的核心思想

DBSCAN 是一种基于密度的聚类算法。它不再像 KMeans 那样假设每个簇都围绕某个中心，而是把“局部样本足够密集的区域”看作一个簇。

### 理解重点

- 只要某些点通过密度关系连成一片，这片区域就可以被当成一个簇。
- 这使 DBSCAN 更适合弯曲形、环形或其他不规则形状的数据。
- 同时，它还能把不属于任何高密度区域的样本标为噪声点。

## 2. `eps` 邻域与点类型

给定参数 `eps` 和 `min_samples`，先定义点 $\mathbf{x}$ 的 `eps` 邻域：

$$
N_{\epsilon}(\mathbf{x}) = \{\mathbf{x}' \in D : d(\mathbf{x}, \mathbf{x}') \leq \epsilon\}
$$

### 核心点

如果一个点的 `eps` 邻域内至少有 `min_samples` 个样本（包含自身），它就是核心点：

$$
|N_{\epsilon}(\mathbf{x})| \geq \text{MinPts}
$$

### 边界点与噪声点

- 边界点：自身不是核心点，但位于某个核心点的 `eps` 邻域内。
- 噪声点：既不是核心点，也不属于任何核心点扩展出的簇。

### 理解重点

- `eps` 决定“邻域有多大”。
- `min_samples` 决定“多密才算高密度区域”。
- 这两个参数必须结合起来理解，单独看其中一个都不够。

## 3. 密度关系如何把点连成簇

### 密度直达

若 $\mathbf{x}$ 是核心点，且 $\mathbf{x}' \in N_{\epsilon}(\mathbf{x})$，则 $\mathbf{x}'$ 从 $\mathbf{x}$ 密度直达。

::: warning 注意
密度直达不对称。一个边界点可以从核心点密度直达，但反过来通常不成立。
:::

### 密度可达

如果存在一条点链 $\mathbf{p}_1, \mathbf{p}_2, \dots, \mathbf{p}_n$，使得每一步都是密度直达，则 $\mathbf{p}_n$ 从 $\mathbf{p}_1$ 密度可达。

### 密度相连

若存在某个点 $\mathbf{o}$，使得 $\mathbf{x}$ 和 $\mathbf{x}'$ 都从 $\mathbf{o}$ 密度可达，则它们密度相连。

### 理解重点

- DBSCAN 实际上是在寻找“能沿密度关系连续扩展的一整片区域”。
- 这就是为什么它不需要显式指定簇中心，也不需要预先给出簇数。
- 只要区域内部足够密、彼此连得起来，它们就会被归为同一簇。

## 4. 簇的定义与噪声的出现

簇 $C$ 可以理解为满足以下性质的点集：

1. 最大性：凡是从簇内点密度可达的点，都属于该簇。
2. 连通性：簇内任意两点都处在同一密度连通结构中。

不属于任何簇的点会被标记为噪声。

### 理解重点

- 对 DBSCAN 来说，“噪声”不是失败，而是算法定义的一部分。
- 在真实数据中，噪声点常常正是我们希望单独识别出来的离群样本。
- 当前源码里，噪声点会通过 `labels_ == -1` 的方式体现出来。

## 5. 参数作用与边界

### 参数速览（本节）

适用参数：`eps`、`min_samples`

| 参数名 | 当前实现取值 | 说明 |
|---|---|---|
| `eps` | `0.3` | 邻域半径，决定多近的点会被看作邻居 |
| `min_samples` | `5` | 核心点判定阈值 |

### 理解重点

- `eps` 太小，很多点会因为邻居不够而变成噪声。
- `eps` 太大，本来分开的区域可能会被错误连成一个簇。
- `min_samples` 越大，算法越保守，越不容易把稀疏区域视为簇。
- 当前源码没有实现 k-距离图选参，因此参数理解应以“解释现有配置为何可用”为主。

## 6. 为什么 DBSCAN 适合当前仓库示例

当前数据来自 `make_moons(...)` 生成的双月牙结构。这类数据的特点是：

- 形状弯曲
- 线性不可分
- 不适合用单一中心去代表整个簇

### 理解重点

- KMeans 在这种数据上往往会沿中心距离把弯月切断。
- DBSCAN 则可以沿月牙形状中的高密度区域不断扩展，因此更容易恢复真实结构。
- 这也是当前仓库把 `make_moons` 作为 DBSCAN 教学数据的原因。

## 常见坑

1. 误以为 DBSCAN 也像 KMeans 一样围绕“簇中心”工作。
2. 只会机械调 `eps`，却不理解它和 `min_samples` 是联动关系。
3. 看到噪声点就认为模型失败。
4. 在密度差异很大的数据上，仍然期待单一 `eps` 同时适配所有簇。

## 小结

- DBSCAN 的核心，是用局部密度关系来定义簇，而不是用中心和平方距离。
- `eps` 和 `min_samples` 决定了核心点、边界点和噪声点的判定方式。
- 当前源码虽然实现简洁，但已经足够展示 DBSCAN 在非球形数据上的优势。
