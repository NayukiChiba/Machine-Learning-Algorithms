# 思路与直觉

> 对应代码：`data_generation/regression.py`、`model_training/regression/regularization.py`
>  
> 相关对象：`RegressionData.regularization()`、`train_model(...)`

## 本章目标

1. 理解为什么普通线性回归在共线性和噪声特征下容易不稳定。
2. 理解 Ridge、Lasso、ElasticNet 各自偏好的系数行为。
3. 把这些直觉和当前仓库构造的数据特征对应起来。

## 重点方法与概念速览

| 名称 | 类型 | 作用 |
|---|---|---|
| Ridge | 模型 | 用 L2 惩罚整体收缩系数 |
| Lasso | 模型 | 用 L1 惩罚推动部分系数精确为 0 |
| ElasticNet | 模型 | 组合 L1 与 L2，兼顾稀疏性与稳定性 |
| `bmi_corr` / `bp_corr` / `s5_corr` | 特征 | 专门用来观察多重共线性下模型行为 |
| `noise_1` ~ `noise_8` | 特征 | 专门用来观察模型是否能抑制无效特征 |

## 1. 为什么当前数据特别适合讲正则化

当前分册不是随便选一个回归数据集，而是刻意构造了两类会让普通最小二乘变难的问题。

### 参数速览（本节）

适用数据特征（本节）：

1. `bmi_corr`、`bp_corr`、`s5_corr`
2. `noise_1` ~ `noise_8`

| 特征类型 | 当前实现 | 会带来的问题 |
|---|---|---|
| 高相关特征 | `*_corr` 来自原始列乘以 `0.9` 再加小噪声 | 系数分配不稳定，容易出现共线性 |
| 纯噪声特征 | `noise_1` ~ `noise_8` 来自随机正态噪声 | 模型容易把无意义波动也当成信号 |

### 示例代码

```python
df["bmi_corr"] = df["bmi"] * 0.9 + rng.normal(scale=0.02, size=len(df))
df["bp_corr"] = df["bp"] * 0.9 + rng.normal(scale=0.02, size=len(df))
df["s5_corr"] = df["s5"] * 0.9 + rng.normal(scale=0.02, size=len(df))

for i in range(self.reg_add_noise_features):
    df[f"noise_{i + 1}"] = rng.normal(size=len(df))
```

### 理解重点

- 如果没有 `*_corr` 特征，三种正则化模型在“系数如何处理共线性”上的差异会弱很多。
- 如果没有 `noise_*` 特征，Lasso 和 ElasticNet 的稀疏化价值就不够直观。
- 也就是说，这份数据本身就是为“观察正则化行为”而设计的。

## 2. 普通线性回归为什么会不稳

普通最小二乘只想把训练误差压小，但不会主动限制系数大小。

### 参数速览（本节）

适用对比对象（本节）：

1. 普通线性回归
2. 正则化回归

| 维度 | 普通线性回归 | 正则化回归 |
|---|---|---|
| 训练目标 | 只关注拟合误差 | 拟合误差 + 系数惩罚 |
| 共线性下表现 | 系数可能剧烈波动 | 系数更稳定或更稀疏 |
| 噪声特征处理 | 容易分到非零权重 | 更倾向压缩无效特征 |
| 可解释性 | 容易受偶然波动影响 | 更容易看出主特征与冗余特征 |

### 理解重点

- 当两个特征几乎表达同一件事时，普通线性回归往往很难稳定决定“权重到底分给谁”。
- 当数据里混入一批无意义噪声特征时，普通线性回归也可能给它们分到看似合理、但其实不稳定的系数。
- 正则化的核心思想，就是主动对系数施加约束，降低这种不稳定性。

## 3. Ridge 的直觉：都保留，但整体缩小

### 参数速览（本节）

适用模型：`Ridge(alpha=2.0, random_state=42)`

| 参数名 | 本例取值 | 说明 |
|---|---|---|
| `alpha` | `2.0` | L2 正则化强度 |
| `random_state` | `42` | 当前源码中显式传入的随机种子 |

### 示例代码

```python
Ridge(alpha=alphas["ridge"], random_state=random_state)
```

### 理解重点

- Ridge 更像是在说：“这些特征也许都还有用，但别让任何一个系数膨胀得太夸张。”
- 面对 `bmi` 和 `bmi_corr` 这类高度相关特征时，Ridge 通常会把权重分摊在同组特征上，而不是只留下一个。
- 因为 L2 惩罚倾向平滑收缩，所以 Ridge 往往保留更多非零系数，但整体数值会更温和。

## 4. Lasso 的直觉：宁可删掉一些特征

### 参数速览（本节）

适用模型：`Lasso(alpha=0.15, max_iter=10000, random_state=42)`

| 参数名 | 本例取值 | 说明 |
|---|---|---|
| `alpha` | `0.15` | L1 正则化强度 |
| `max_iter` | `10000` | 坐标下降法的最大迭代次数 |
| `random_state` | `42` | 当前源码中显式传入的随机种子 |

### 示例代码

```python
Lasso(alpha=alphas["lasso"], max_iter=10000, random_state=random_state)
```

### 理解重点

- Lasso 的偏好更强硬：如果某些特征贡献不稳定或不明显，它会倾向直接把系数压到 0。
- 在当前数据里，最值得观察的就是 `noise_1` ~ `noise_8` 是否被压到接近 0。
- 对于高度相关的一组特征，Lasso 也常常只保留其中一部分，因此可解释性更强，但稳定性未必总比 Ridge 更好。

## 5. ElasticNet 的直觉：既想筛选，也想保持稳定

### 参数速览（本节）

适用模型：`ElasticNet(alpha=0.2, l1_ratio=0.5, max_iter=10000, random_state=42)`

| 参数名 | 本例取值 | 说明 |
|---|---|---|
| `alpha` | `0.2` | 总体正则化强度 |
| `l1_ratio` | `0.5` | L1 比例，`0` 更像 Ridge，`1` 更像 Lasso |
| `max_iter` | `10000` | 最大迭代次数 |
| `random_state` | `42` | 当前源码中显式传入的随机种子 |

### 示例代码

```python
ElasticNet(
    alpha=alphas["elasticnet"],
    l1_ratio=l1_ratio,
    max_iter=10000,
    random_state=random_state,
)
```

### 理解重点

- ElasticNet 试图在“筛掉无效特征”和“稳定处理相关特征”之间找平衡。
- 在当前这份既有 `*_corr` 又有 `noise_*` 的数据里，它通常比纯 Lasso 更稳，比纯 Ridge 更容易产生稀疏效果。
- `l1_ratio=0.5` 表示当前实现并没有极端偏向某一边，而是用一个中间配置来展示折中效果。

## 6. 如何把直觉映射到当前训练日志

当前训练函数不会只返回模型对象，还会打印每个模型的系数信息。

### 参数速览（本节）

适用输出项（本节）：

1. `接近 0 的系数数量`
2. 各特征系数打印

| 观察项 | 可以帮助判断什么 |
|---|---|
| 接近 0 的系数数量 | 模型是否明显产生了稀疏化 |
| `noise_*` 的系数 | 模型是否成功抑制无效特征 |
| `bmi` 与 `bmi_corr` 等同组系数 | 模型如何处理共线性 |

### 理解重点

- 如果一个模型把很多 `noise_*` 系数压到接近 0，说明它在做特征筛选。
- 如果一个模型让 `bmi` 与 `bmi_corr` 都保留非零系数，说明它更偏向“共享权重”而不是“只选一个”。
- 当前分册真正想让你观察的，不只是预测结果，还有这些系数结构差异。

## 常见坑

1. 把“Lasso 会产生 0 系数”误解成“Lasso 一定预测更好”。
2. 只看某一个特征系数大小，不结合 `*_corr` 整组特征一起看。
3. 只看稀疏性，不看残差图中的整体预测表现。

## 小结

- 正则化回归要解决的核心问题，是共线性和噪声特征带来的系数不稳定与过拟合风险。
- Ridge 更像整体收缩，Lasso 更像主动筛选，ElasticNet 则在两者之间折中。
- 当前仓库这份数据正是为了把这三种行为差异尽量清楚地暴露出来。
