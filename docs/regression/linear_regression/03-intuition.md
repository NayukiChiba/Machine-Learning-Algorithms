# 思路与直觉

> 对应代码：`data_generation/regression.py`、`model_training/regression/linear_regression.py`
>  
> 相关对象：`RegressionData.linear_regression()`、`train_model(...)`

## 本章目标

1. 理解线性回归为什么适合用来拟合连续值目标。
2. 理解系数正负、大小和截距在当前数据里的直观含义。
3. 把这些直觉和当前仓库的房价合成数据对应起来。

## 重点方法与概念速览

| 名称 | 类型 | 作用 |
|---|---|---|
| `LinearRegression` | 模型 | 用一个线性函数拟合连续值目标 |
| `coef_` | 属性 | 表示各特征对预测值的线性影响 |
| `intercept_` | 属性 | 表示所有特征取零时的基线预测 |
| `面积`、`房间数`、`房龄` | 特征 | 当前数据中最直接可解释的输入变量 |

## 1. 线性回归想做什么

线性回归的核心目标，是用一条直线或一个超平面去近似连续值目标和输入特征之间的关系。

### 参数速览（本节）

适用关系（本节）：

1. `y = w1 * x1 + w2 * x2 + ... + b`

| 项目 | 当前含义 |
|---|---|
| 输入 | 多个连续特征 |
| 输出 | 一个连续数值目标 |
| 学习结果 | 一组系数 + 一个截距 |
| 预测方式 | 线性加权求和 |

### 理解重点

- 如果数据关系本身接近线性，线性回归通常能用非常简单的形式给出不错的拟合。
- 相比更复杂的非线性模型，线性回归最大的优势是结果容易解释。
- 当前仓库这份数据就是专门为了突出这种“简单但可解释”的特性而构造的。

## 2. 当前数据为什么特别适合讲直觉

### 参数速览（本节）

适用真实关系（本节）：

1. `price = 2 * 面积 + 10 * 房间数 - 3 * 房龄 + noise + 50`

| 特征 | 真实影响 |
|---|---|
| `面积` | 正向影响，系数为 `2` |
| `房间数` | 正向影响，系数为 `10` |
| `房龄` | 负向影响，系数为 `-3` |
| 截距 | 基线项为 `50` |

### 示例代码

```python
price = (
    2 * area
    + 10 * num
    - 3 * age
    + rng.normal(loc=0, scale=self.lr_noise, size=self.n_samples)
    + 50
)
```

### 理解重点

- 这份数据的优势不在于“复杂”，而在于“真实关系清楚”。
- 训练完成后，只要看 `coef_` 和 `intercept_`，就能直观判断模型有没有学到正确方向。
- 这类数据非常适合初学时建立“系数到底表示什么”的直觉。

## 3. 如何理解系数的正负和大小

### 参数速览（本节）

适用属性：`model.coef_`

| 系数现象 | 直观含义 |
|---|---|
| 系数为正 | 特征增大时，预测值倾向增大 |
| 系数为负 | 特征增大时，预测值倾向减小 |
| 系数绝对值更大 | 在同量纲前提下，影响通常更明显 |

### 理解重点

- 在当前数据里，`面积` 和 `房间数` 的系数应该偏正，`房龄` 的系数应该偏负。
- `房间数` 的真实系数是 `10`，比 `面积` 的 `2` 更大，说明它对价格的单位变化影响更强。
- 但训练结果不会完全等于真实公式，因为数据里还叠加了噪声项。

## 4. 如何理解截距

### 参数速览（本节）

适用属性：`model.intercept_`

| 项目 | 当前含义 |
|---|---|
| `intercept_` | 所有特征取 0 时的基线预测 |
| 当前真实值 | 理论上接近 `50` |

### 理解重点

- 截距不是“某个特征的重要程度”，而是整个线性关系的基准起点。
- 在当前公式里，截距对应 `+50` 这一常数项。
- 因为样本带噪声、训练只看到有限数据，所以学到的截距通常只会接近而不是精确等于 `50`。

## 5. 与更复杂回归方法相比，线性回归的优势和边界

### 参数速览（本节）

适用对比对象（本节）：

1. 线性回归
2. 非线性回归方法

| 维度 | 线性回归 | 非线性方法 |
|---|---|---|
| 可解释性 | 很强 | 通常更弱 |
| 拟合复杂关系能力 | 较弱 | 通常更强 |
| 训练与使用成本 | 低 | 往往更高 |
| 当前分册重点 | 理解系数和误差 | 不在本章重点内 |

### 理解重点

- 线性回归不是为了处理所有复杂模式，而是为了在“关系接近线性”时提供简单、稳定、可解释的基线模型。
- 当数据关系明显非线性时，像 SVR、决策树这类方法通常更合适。
- 当前仓库先讲线性回归，就是为了建立最基础的回归建模直觉。

## 6. 直觉如何映射到当前训练日志

### 参数速览（本节）

适用输出项（分项）：

1. `截距(intercept)`
2. `斜率(coefficients)`

| 日志项 | 可以帮助判断什么 |
|---|---|
| 截距 | 是否接近真实基线项 |
| 各特征系数 | 是否学到正确方向和大致大小 |

### 示例代码

```python
print(f"截距(intercept): {model.intercept_:.2f}")
print("斜率(coefficients):")
for name, coef in zip(names, model.coef_):
    print(f"  {name}: {coef:.2f}")
```

### 理解重点

- 当前训练函数最关键的输出，不是复杂指标，而是截距和各特征系数。
- 如果 `房龄` 的系数被学成明显正数，就说明模型结果和真实关系不一致，需要回头看数据或样本波动。
- 这也是为什么本分册强调“先看关系方向，再看误差图”。

## 常见坑

1. 把系数大小直接拿来跨不同量纲特征比较，而不考虑单位影响。
2. 把截距误解成“某个特征的权重”。
3. 看到训练结果和真实公式不完全一致，就误以为模型出错，忽略了噪声项和有限样本带来的偏差。

## 小结

- 线性回归最核心的直觉，是用一个线性函数去近似连续值目标。
- 当前仓库这份房价合成数据非常适合用来理解系数正负、大小和截距的含义。
- 把这些直觉建立起来之后，再看数学原理、训练流程和评估图像会更容易。
