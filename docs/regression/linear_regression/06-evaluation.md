# 评估与诊断

> 对应代码：`pipelines/regression/linear_regression.py`、`result_visualization/residual_plot.py`、`result_visualization/learning_curve.py`
>  
> 相关对象：`plot_residuals(...)`、`plot_learning_curve(...)`

## 本章目标

1. 明确当前仓库实际使用了哪些评估手段，而不是泛泛讨论所有回归指标。
2. 理解残差图和学习曲线分别能帮助我们诊断什么。
3. 明确当前实现没有做哪些数值指标输出，以免误读源码能力边界。

## 重点方法与概念速览

| 名称 | 类型 | 作用 |
|---|---|---|
| `plot_residuals(...)` | 函数 | 生成预测-真实图和残差分布图 |
| `plot_learning_curve(...)` | 函数 | 生成训练得分和验证得分曲线 |
| `residuals = y_true - y_pred` | 派生量 | 衡量每个样本的预测误差 |
| `scoring='r2'` | 参数 | 当前学习曲线使用的评分指标 |

## 1. 当前实现真正做了什么评估

### 参数速览（本节）

适用评估手段（本节）：

1. 残差图
2. 学习曲线

| 评估方式 | 来源 | 用途 |
|---|---|---|
| 残差图 | `plot_residuals(...)` | 观察测试集上的整体拟合情况与误差分布 |
| 学习曲线 | `plot_learning_curve(...)` | 观察训练样本数变化时训练/验证得分走势 |

### 理解重点

- 当前 linear regression 流水线没有显式打印 `MSE`、`MAE`、`RMSE`、`R^2` 等数值指标。
- 这并不表示这些指标不重要，而是说明本仓库当前实现更强调图像化诊断。
- 因此阅读这一分册时，不能把“指标表格”想成已经在源码里实现的内容。

## 2. 残差图是怎么生成的

### 参数速览（本节）

适用函数：`plot_residuals(y_true, y_pred, title='残差分析', dataset_name='default', model_name='model', figsize=(14, 5))`

| 参数名 | 本例取值 | 说明 |
|---|---|---|
| `y_true` | `y_test` | 测试集真实值 |
| `y_pred` | 模型预测值 | 测试集预测结果 |
| `dataset_name` | `"linear_regression"` | 输出目录名 |
| `model_name` | `"linear_regression"` | 输出文件名前缀 |
| `figsize` | `(14, 5)` | 图像尺寸 |

### 示例代码

```python
residuals = y_true - y_pred

ax1.scatter(y_true, y_pred, alpha=0.6, edgecolors="k", linewidth=0.5, s=30)
ax2.scatter(y_pred, residuals, alpha=0.6, edgecolors="k", linewidth=0.5, s=30)
ax2.axhline(y=0, color="r", linestyle="--", linewidth=1.5)
```

### 理解重点

- 第一张子图看“预测值是否靠近真实值对角线”，适合粗看拟合效果。
- 第二张子图看“残差是否围绕 0 随机分布”，适合发现系统偏差、异常点和异方差线索。
- 这两张图组合起来，是当前流水线最直接的测试集诊断方式。

## 3. 学习曲线是怎么生成的

### 参数速览（本节）

适用函数：`plot_learning_curve(model, X, y, cv=5, scoring='accuracy', train_sizes=None, ...)`

| 参数名 | 本例取值 | 说明 |
|---|---|---|
| `model` | `LinearRegression()` | 传入一个新的未训练模型实例 |
| `X` | `X_train` | 使用训练集特征 |
| `y` | `y_train` | 使用训练集标签 |
| `cv` | 默认 `5` | 交叉验证折数 |
| `scoring` | `"r2"` | 当前线性回归分册使用的评分指标 |
| `train_sizes` | 默认 `0.1` 到 `1.0` 共 10 个点 | 训练样本比例 |

### 示例代码

```python
plot_learning_curve(
    LinearRegression(),
    X_train,
    y_train,
    scoring="r2",
    title="线性回归 学习曲线",
    dataset_name=DATASET,
    model_name=MODEL,
)
```

### 理解重点

- 学习曲线内部调用的是 `sklearn.model_selection.learning_curve(...)`，会在不同训练样本规模下重复训练和验证。
- 当前分册使用的是 `R^2` 作为学习曲线评分指标，但这个分数不会单独打印成一张文本表。
- 学习曲线关注的是“样本量变化趋势”，而不是某一次测试集预测结果。

## 4. 看残差图时重点观察什么

### 参数速览（本节）

适用观察点（本节）：

1. 对角线贴合程度
2. 残差围绕 0 的分布
3. 离群点与扇形结构

| 现象 | 可能含义 |
|---|---|
| 点大多贴近 `y=x` | 整体预测较稳定 |
| 残差围绕 0 随机散开 | 没有明显系统偏差 |
| 残差整体偏正或偏负 | 模型存在系统性高估或低估 |
| 残差呈扇形扩散 | 可能存在异方差或局部拟合不足 |
| 少量极端散点 | 可能有异常样本或噪声影响较大 |

### 理解重点

- 线性回归虽然结构简单，但残差图依然可以暴露很多问题。
- 如果残差随着预测值明显呈曲线趋势，往往说明线性假设本身不够好。
- 当前数据是人为构造的近线性关系，因此残差图通常会比较规整，也更适合作为基线示例。

## 5. 看学习曲线时重点观察什么

### 参数速览（本节）

适用观察点（本节）：

1. 训练得分走势
2. 验证得分走势
3. 两条曲线之间的间距

| 现象 | 可能含义 |
|---|---|
| 训练和验证得分都较低 | 可能欠拟合 |
| 训练得分高、验证得分低 | 可能过拟合 |
| 两条曲线逐渐收敛 | 模型随样本数增加趋于稳定 |
| 曲线波动很大 | 数据量可能偏少或噪声影响较强 |

### 理解重点

- 学习曲线不是直接判断“预测准不准”，而是帮助判断模型复杂度是否合适、样本量是否足够。
- 当前线性回归模型较简单，学习曲线通常不会像高复杂度模型那样出现非常夸张的训练/验证分裂。
- 如果后续你提高 `lr_noise` 或减少样本数，这部分图形变化会更明显。

## 6. 当前实现没有做什么

### 参数速览（本节）

当前源码未包含的内容：

1. 显式数值指标打印
2. 自动调参
3. 交叉验证结果表格输出

| 未实现项 | 当前状态 |
|---|---|
| `MSE` / `MAE` / `RMSE` / `R^2` 打印 | 未在流水线中出现 |
| 超参数搜索 | 未使用 `GridSearchCV` 等 API |
| 学习曲线明细表 | 未单独输出各折分数表 |

### 理解重点

- 评估章节必须以源码为准，不能把“理论上常用指标”写成“当前仓库已经实现”。
- 当前实现的评估重点是图形诊断，而不是数值面板。
- 如果后续扩展这部分，最自然的方向是补指标打印，而不是替换现有残差图和学习曲线逻辑。

## 常见坑

1. 只看残差图，不看学习曲线，错过对样本量与泛化趋势的观察。
2. 只看学习曲线，不看测试集残差图，错过对具体预测误差分布的观察。
3. 误以为当前流水线已经输出了完整的 `R^2`、`MSE` 指标表，实际源码并没有这一步。

## 小结

- 当前线性回归的评估主线由两部分组成：残差图和学习曲线。
- 残差图负责看测试误差分布，学习曲线负责看训练样本规模变化下的泛化趋势。
- 只有把这两条线索一起看，才能更完整地理解当前实现的表现。
