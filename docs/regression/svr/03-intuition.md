# 思路与直觉

> 对应代码：`model_training/regression/svr.py`、`pipelines/regression/svr.py`
>  
> 相关对象：`train_model(...)`、`model.support_`

## 本章目标

1. 从本仓库数据与参数设置理解 SVR 为什么适合当前任务。
2. 通过 `C`、`epsilon`、`gamma` 的作用形成调参直觉。
3. 理解“支持向量数量”在代码输出中的诊断意义。

## 重点概念速览

| 名称 | 类型 | 作用 |
|---|---|---|
| `kernel='rbf'` | 默认配置 | 让模型能够拟合非线性关系 |
| `C=10.0` | 默认配置 | 对管道外误差施加较强惩罚 |
| `epsilon=0.1` | 默认配置 | 只容忍较小的无损失误差范围 |
| `gamma='scale'` | 默认配置 | 自动根据特征方差调整核函数尺度 |
| `model.support_` | 属性 | 反映模型复杂度的重要信号 |

## 1. 为什么当前任务适合 SVR

本仓库的 `svr_data` 来自 Friedman1 数据。这个数据集的目标函数是明显非线性的，因此线性模型往往很难完全捕捉其结构，而 SVR 可以借助核函数在更高维空间中拟合这种非线性关系。

### 理解重点

- 如果数据关系明显弯曲，RBF 核往往比线性回归更自然。
- SVR 同时保留了“误差容忍”机制，不会像某些模型那样对每个小误差都强烈反应。

## 2. 从默认参数建立第一层直觉

`train_model(...)` 的默认设置是：

- `kernel='rbf'`
- `C=10.0`
- `epsilon=0.1`
- `gamma='scale'`

### 参数速览（本节）

适用参数（分项）：

1. `C`
2. `epsilon`
3. `gamma`
4. `model.support_`

| 参数名 | 本例取值 | 说明 |
|---|---|---|
| `C` | `10.0` | 增大时更贴合训练集，可能降低偏差但提高方差 |
| `epsilon` | `0.1` | 增大时管道更宽，模型更平滑但可能欠拟合 |
| `gamma` | `'scale'` | 值越大时单个样本影响越局部 |
| 返回值（`model.support_`） | 索引数组 | 支持向量越多，通常说明模型越复杂 |

### 理解重点

- 当前默认参数适合“中等规模、非线性明显、需要较细致拟合”的回归问题。
- 这组参数不是绝对最优，但足够作为稳定的教学和实验起点。

## 3. 与其它回归方法的直观对比

| 方法 | 拟合能力 | 对尺度敏感性 | 当前仓库中对应实现 |
|---|---|---|---|
| 线性回归 | 线性关系 | 中等 | `pipelines.regression.linear_regression` |
| 决策树回归 | 非线性（分段） | 低 | `pipelines.regression.decision_tree` |
| SVR（RBF） | 非线性（核映射） | 高 | `pipelines.regression.svr` |

### 理解重点

- 对 `svr_data` 这种非线性数据，SVR 通常比纯线性模型更容易得到稳定拟合。
- 但代价是它对特征尺度更敏感，因此标准化不能省略。

## 4. 支持向量数量为什么重要

训练函数最后会执行：

```python
n_sv = model.support_.shape[0]
print(f"支持向量数量: {n_sv}")
```

### 理解重点

- 支持向量数量是模型复杂度的一个直观信号。
- 如果支持向量非常多，往往意味着模型对训练样本依赖更强，泛化风险也可能升高。
- 因此当前源码在训练结束后立刻打印这一信息，是很有价值的诊断设计。

## 5. 第一轮调参时看什么

训练函数会打印：

1. `kernel`、`C`、`epsilon`、`gamma`
2. 当 `kernel='poly'` 时，还会打印 `degree` 和 `coef0`
3. `支持向量数量`
4. `SVR 训练耗时`

### 理解重点

- 第一轮调参时，先看这几项日志，再去看残差图和学习曲线，通常最有效。
- 如果残差图出现系统性模式，往往说明核函数或参数设置不匹配。

## 常见坑

1. 只改 `C`，忽略 `epsilon` 和 `gamma` 的联动影响。
2. 只看训练结果，不看支持向量数量和残差结构。
3. 误以为 SVR 可以像决策树那样直接处理任意尺度特征。

## 小结

- 当前仓库默认的 SVR 配置，核心直觉是“用 RBF 核拟合非线性，同时通过 `C` 和 `epsilon` 控制复杂度”。
- 调参时最值得优先观察的不是某一个单独数字，而是“参数打印 + 支持向量数量 + 图形诊断”的组合。
