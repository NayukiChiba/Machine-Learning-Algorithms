# 思路与直觉

> 对应代码：`data_generation/classification.py`、`pipelines/classification/knn.py`
>
> 对比对象：`docs/classification/logistic_regression/`

## 本章目标

1. 用直观方式理解 KNN 到底在做什么。
2. 理解为什么它在当前双月牙数据上更能体现局部分类优势。
3. 理解它与更偏全局边界的方法在思路上的关键差异。

## 重点方法与概念速览

| 名称 | 类型 | 作用 |
|---|---|---|
| 邻域投票 | 核心直觉 | 用最近样本的类别决定预测结果 |
| 局部结构 | 决策依据 | 当前点周围的几何关系比全局边界更重要 |
| 非线性边界 | 决策形状 | 当前双月牙数据下更自然的分类方式 |
| `k` 值 | 超参数 | 控制“看多大范围的邻居” |
| 逻辑回归 | 对比算法 | 更偏全局线性边界 |

## 1. 为什么需要 KNN

对于一些分类问题，我们不一定非要先学一条全局边界。有时更自然的问题是：

一个新样本周围都是什么类别的点？

KNN 给出的思路是：

- 不急着先学参数。
- 先找到离当前样本最近的若干训练样本。
- 再根据这些邻居的类别来投票决定结果。

### 理解重点

- 这使 KNN 成为一种非常典型的局部方法。
- 它的预测不依赖一个全局固定公式，而依赖当前点周围的数据分布。
- 这也是为什么它常被称为懒惰学习模型。

## 2. 为什么当前仓库示例里它表现合理

当前 KNN 数据来自 `make_moons(...)`，特点是：

- 双月牙形状
- 类别边界弯曲
- 全局线性边界不自然

### 理解重点

- 对这类数据来说，“局部邻域关系”通常比“全局一条直线怎么切”更重要。
- 如果附近大多数点都属于同一类，那么当前点也很可能属于这一类。
- 这正是 KNN 在当前数据上能体现优势的原因。

## 3. 用“看周围邻居是谁”理解算法

可以把 KNN 理解成：

1. 找到待预测点附近的若干个邻居。
2. 看这些邻居的类别分布。
3. 由它们投票决定最终类别。

### 示例代码

```text
先找最近的 K 个点
再统计这些点分别属于哪一类
最后根据投票结果给出预测
```

### 理解重点

- 这也是为什么 `n_neighbors` 在当前分册里最值得专门观察。
- 如果把 KNN 仅理解成“一个会分类的黑盒”，就会错过它最有辨识度的局部投票思想。

## 4. 为什么标准化会显著影响结果

当前流水线里训练前有这样一步：

```python
scaler = StandardScaler()
X_train_s = scaler.fit_transform(X_train)
X_test_s = scaler.transform(X_test)
```

### 理解重点

- KNN 完全依赖距离关系，因此对特征尺度特别敏感。
- 如果一个特征量级远大于另一个特征，它就会主导“最近邻”判断。
- 这也是为什么 KNN 比很多线性模型更需要把标准化写得明确、讲得透彻。

## 5. 与逻辑回归的直觉差异

两者的核心差异可以这样理解：

| 算法 | 更关注什么 | 核心依据 |
|---|---|---|
| LogisticRegression | 全局线性边界与概率输出 | 线性打分 + Sigmoid |
| KNN | 局部邻域关系 | 最近邻投票 |

### 理解重点

- 逻辑回归更像是在问：整张图上最好的一条全局边界是什么。
- KNN 更像是在问：当前点周围的邻居大多是什么类别。
- 当前双月牙数据更适合把 KNN 作为局部分类基线来讲解。

## 常见坑

1. 只知道 KNN 很简单，却说不出它为什么适合当前双月牙数据。
2. 把所有样本都当成同等重要，而忽略 KNN 只关心局部邻域。
3. 只会机械调 `k`，却不理解它对应的是局部范围大小。
4. 忽略标准化，让近邻关系本身失真。

## 小结

- KNN 的直觉重点有两个：局部邻域和投票决策。
- 当前仓库使用双月牙数据，正好体现了它在非线性局部结构上的优势。
- 如果已经理解“为什么它先看附近点是谁，再决定分类”，就已经抓住了本分册最核心的直觉。
