# 数学原理

> 对应代码：`model_training/classification/svc.py`、`pipelines/classification/svc.py`
>
> 对应对象：`SVC(...)`、`model.fit(X_train, y_train)`、`model.n_support_`

## 本章目标

1. 理解 SVC 为什么要寻找最大间隔分类边界。
2. 理解 `C`、核函数和 RBF 核在当前实现中的作用。
3. 理解为什么当前同心圆数据必须依赖非线性核方法。

## 重点方法与概念速览

| 名称 | 类型 | 作用 |
|---|---|---|
| 最大间隔 | 优化目标 | 寻找泛化能力更强的分类边界 |
| 支持向量 | 核心样本 | 真正决定分类边界的位置 |
| 软间隔 | 机制 | 允许少量违例样本，提升鲁棒性 |
| `C` | 超参数 | 控制间隔大小与误分类惩罚的权衡 |
| 核函数 | 技巧 | 把非线性分类问题转化为高维线性可分问题 |
| RBF 核 | 核函数 | 当前源码默认使用的核函数 |

## 1. SVC 的核心思想

SVC 寻找一个最大间隔超平面，将不同类别的数据尽可能分开。直观上，它不只关心“能不能分开”，还关心“分开的边界离两边样本有多远”。

### 超平面形式

$$
\mathbf{w}^T \mathbf{x} + b = 0
$$

### 理解重点

- 在所有可行分类边界里，间隔越大，模型通常越稳健。
- SVC 的思路不是简单追求训练集 100% 正确，而是寻找更有泛化能力的边界。
- 当前文档里的“最大间隔”是理解支持向量机的第一原则。

## 2. 硬间隔与软间隔

当数据线性可分时，可以写出硬间隔优化问题：

$$
\min_{\mathbf{w}, b} \frac{1}{2} \|\mathbf{w}\|^2
\quad \text{s.t.} \quad y_i(\mathbf{w}^T \mathbf{x}_i + b) \geq 1
$$

但真实数据通常会有噪声，因此更常见的是软间隔形式：

$$
\min_{\mathbf{w}, b, \boldsymbol{\xi}} \frac{1}{2} \|\mathbf{w}\|^2 + C \sum_{i=1}^{N} \xi_i
$$

$$
\text{s.t.} \quad y_i(\mathbf{w}^T \mathbf{x}_i + b) \geq 1 - \xi_i, \quad \xi_i \geq 0
$$

### 理解重点

- `C` 越大，模型越不愿意容忍误分类，边界会更努力贴合训练数据。
- `C` 越小，模型越强调更宽的间隔，对少量违例更宽容。
- 当前源码默认 `C=1.0`，这是一个兼顾稳定性和教学简洁度的常见起点。

## 3. 为什么支持向量最重要

在 SVC 中，真正决定分类边界的，并不是所有训练样本，而是靠近边界的那部分关键样本，也就是支持向量。

### KKT 条件对应的直觉

只有落在间隔边界上或违反间隔约束的样本，才会真正影响最终解。

### 理解重点

- 这也是为什么当前训练日志会特别打印 `model.n_support_`。
- `n_support_` 能帮助读者观察：模型到底依赖了多少关键样本来确定边界。
- 对教学场景来说，支持向量数量是比单纯打印“训练完成”更有解释力的结果。

## 4. 核函数为什么能处理非线性问题

对偶形式中只涉及样本内积，因此可以把内积替换成核函数：

| 核函数 | 公式 |
|---|---|
| 线性核 | $K(\mathbf{x}, \mathbf{z}) = \mathbf{x}^T \mathbf{z}$ |
| 多项式核 | $K(\mathbf{x}, \mathbf{z}) = (\gamma \mathbf{x}^T \mathbf{z} + r)^d$ |
| RBF 核 | $K(\mathbf{x}, \mathbf{z}) = \exp\left(-\gamma \|\mathbf{x} - \mathbf{z}\|^2\right)$ |
| Sigmoid 核 | $K(\mathbf{x}, \mathbf{z}) = \tanh(\gamma \mathbf{x}^T \mathbf{z} + r)$ |

### 理解重点

- 核技巧的作用，是让模型不用显式构造高维特征，也能获得非线性边界。
- 当前源码虽然在注释里列出了多种核函数，但真正默认使用的是 `rbf`。
- 因此本分册的理论重点，不应平均分配给所有核，而应落在 RBF 核上。

## 5. 为什么当前实现默认使用 RBF 核

当前训练代码里使用的是：

```python
SVC(C=1.0, kernel="rbf", gamma="scale", random_state=42)
```

RBF 核形式为：

$$
K(\mathbf{x}, \mathbf{z}) = \exp\left(-\gamma \|\mathbf{x} - \mathbf{z}\|^2\right)
$$

### 理解重点

- RBF 核本质上用“距离越近，相似度越高”的方式构造非线性决策能力。
- 当前数据是同心圆，线性超平面无法把内圈和外圈正确分开。
- RBF 核恰好适合这种“局部相似性比全局线性更重要”的数据形态。

## 6. 为什么标准化会显著影响 SVC

### 理解重点

- 当前模型是基于距离和内积运作的核方法，对特征尺度非常敏感。
- 如果不标准化，某一维数值范围更大，就会主导核函数中的距离计算。
- 这也是当前流水线必须在训练前执行 `StandardScaler` 的原因之一。

## 常见坑

1. 只知道 SVM 是“分类器”，却不知道它真正优化的是最大间隔。
2. 忽略 `C` 对间隔与误分类权衡的作用。
3. 在明显非线性的数据上，仍然默认按线性边界理解模型行为。
4. 忽略标准化，让核函数的距离含义失真。

## 小结

- SVC 的核心，是通过最大间隔思想找到泛化能力更强的分类边界。
- `C` 决定软间隔权衡，支持向量决定最终边界位置。
- 当前源码默认使用 RBF 核，是因为它与同心圆这类非线性数据高度匹配。
