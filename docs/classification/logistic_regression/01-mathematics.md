# 数学原理

> 对应代码：`model_training/classification/logistic_regression.py`、`pipelines/classification/logistic_regression.py`
>
> 对应对象：`LogisticRegression(...)`、`model.fit(X_train, y_train)`、`model.coef_`、`model.intercept_`

## 本章目标

1. 理解逻辑回归为什么虽然叫“回归”，本质上却是分类模型。
2. 理解 Sigmoid、对数几率、交叉熵和梯度在当前实现中的作用。
3. 理解 `coef_`、`intercept_` 与线性决策边界之间的关系。

## 重点方法与概念速览

| 名称 | 类型 | 作用 |
|---|---|---|
| 线性得分 | 模型部分 | 先对输入特征做线性组合 |
| Sigmoid 函数 | 概率映射 | 把线性得分映射到 `(0, 1)` 概率区间 |
| 对数几率 | 解释方式 | 把概率分类问题转成线性表达 |
| 交叉熵损失 | 优化目标 | 衡量预测概率与真实标签差异 |
| 梯度 | 优化信息 | 决定参数如何更新 |
| 正则化与 `C` | 超参数机制 | 控制模型复杂度与拟合强度 |

## 1. 逻辑回归的核心思想

逻辑回归先计算一个线性得分，再把这个得分压缩成概率输出。因此它虽然名字里有“回归”，但最终目标是做分类判断。

### 线性部分

$$
z = \mathbf{w}^T \mathbf{x} + b = \sum_{j=1}^{d} w_j x_j + b
$$

### Sigmoid 函数

$$
\sigma(z) = \frac{1}{1 + e^{-z}}
$$

### 概率输出

$$
P(y=1 \mid \mathbf{x}) = \sigma(\mathbf{w}^T \mathbf{x} + b)
$$

### 理解重点

- 当前模型不是直接输出“正类/负类”，而是先输出正类概率。
- 这也是为什么当前流水线可以直接调用 `predict_proba(...)` 来绘制 ROC 曲线。
- 逻辑回归的一个核心优势，就是分类结果和概率解释可以天然结合。

## 2. 对数几率为什么是线性的

逻辑回归对对数几率建模为线性函数：

$$
\ln \frac{P(y=1 \mid \mathbf{x})}{P(y=0 \mid \mathbf{x})} = \mathbf{w}^T \mathbf{x} + b
$$

这意味着决策边界：

$$
\mathbf{w}^T \mathbf{x} + b = 0
$$

### 理解重点

- 一旦概率阈值取 `0.5`，分类边界就对应线性得分为 0 的超平面。
- 这也是为什么 `coef_` 和 `intercept_` 在逻辑回归里很有解释价值。
- 当前数据如果近线性可分，逻辑回归就能自然形成稳定边界。

## 3. 极大似然与交叉熵损失

对训练集 $\{(\mathbf{x}_i, y_i)\}_{i=1}^{N}$，似然函数为：

$$
L(\mathbf{w}, b) = \prod_{i=1}^{N} \hat{p}_i^{\,y_i} (1 - \hat{p}_i)^{1 - y_i}
$$

其中 $\hat{p}_i = \sigma(\mathbf{w}^T \mathbf{x}_i + b)$。

取对数并加负号后，得到要最小化的交叉熵损失：

$$
\mathcal{L}(\mathbf{w}, b) = -\frac{1}{N} \sum_{i=1}^{N} \Big[ y_i \ln \hat{p}_i + (1 - y_i) \ln (1 - \hat{p}_i) \Big]
$$

### 理解重点

- 逻辑回归不是靠最小二乘，而是靠极大似然或等价的交叉熵目标来训练。
- 这使它更适合二分类概率建模，而不是连续值回归。
- 当前数学章节应明确这一点，避免和线性回归的损失函数混淆。

## 4. 梯度为什么长这样

交叉熵对参数 $w_j$ 的偏导数为：

$$
\frac{\partial \mathcal{L}}{\partial w_j} = \frac{1}{N} \sum_{i=1}^{N} (\hat{p}_i - y_i) \, x_{ij}
$$

向量形式：

$$
\nabla_{\mathbf{w}} \mathcal{L} = \frac{1}{N} \mathbf{X}^T (\hat{\mathbf{p}} - \mathbf{y})
$$

### 理解重点

- 这个梯度形式非常直观：预测概率与真实标签的误差，乘上对应特征值，再汇总起来更新参数。
- 也可以把它理解为“当前模型在哪些方向上高估或低估了正类概率”。
- 当前源码虽然没有手写梯度下降，但理解这层数学仍然有助于理解优化器与训练行为。

## 5. 正则化与 `C`

当前训练代码里使用的是：

```python
LogisticRegression(
    penalty="l2",
    C=1.0,
    solver="lbfgs",
    max_iter=1000,
    class_weight=None,
    random_state=42,
)
```

### 理解重点

- `penalty='l2'` 表示当前模型使用 L2 正则化。
- `C` 是正则化强度的倒数，`C` 越大，正则越弱；`C` 越小，正则越强。
- 这是当前逻辑回归分册最容易被写反的地方，文档必须明确说明。

## 6. 为什么标准化会影响训练与解释

### 理解重点

- 标准化会影响优化器收敛速度，也会影响 `coef_` 的可比性。
- 在不同特征尺度差异很大时，不标准化会让某些维度主导训练过程。
- 这也是当前流水线必须在训练前执行 `StandardScaler` 的原因之一。

## 7. 多分类扩展如何看待

逻辑回归可以扩展到 Softmax 多分类：

$$
P(y = k \mid \mathbf{x}) = \frac{e^{\mathbf{w}_k^T \mathbf{x} + b_k}}{\sum_{j=1}^{K} e^{\mathbf{w}_j^T \mathbf{x} + b_j}}
$$

### 理解重点

- 当前数学章节可以保留这一扩展，帮助建立完整视角。
- 但当前工程实现使用的是二分类数据，因此数据、模型、训练、评估章节都应聚焦当前二分类逻辑回归流程，而不是把重点转移到多分类实现。

## 常见坑

1. 把逻辑回归误当成线性回归加阈值。
2. 忽略交叉熵与极大似然的关系。
3. 把 `C` 的含义写反。
4. 忽略标准化对优化和系数解释的影响。

## 小结

- 逻辑回归的核心，是把线性得分通过 Sigmoid 转成概率，再用交叉熵目标进行训练。
- `coef_` 与 `intercept_` 直接决定线性决策边界位置。
- 当前源码默认使用 L2 正则化二分类逻辑回归，这和当前高维近线性可分数据高度匹配。
