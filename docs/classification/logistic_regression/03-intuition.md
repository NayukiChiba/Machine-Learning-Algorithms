# 思路与直觉

> 对应代码：`data_generation/classification.py`、`pipelines/classification/logistic_regression.py`
>
> 对比对象：`docs/classification/svc/`

## 本章目标

1. 用直观方式理解逻辑回归到底在做什么。
2. 理解为什么它在当前高维二分类数据上是合理基线。
3. 理解它与更强调边界复杂度的方法在思路上的关键差异。

## 重点方法与概念速览

| 名称 | 类型 | 作用 |
|---|---|---|
| 线性打分 | 核心直觉 | 先给样本一个线性分值 |
| Sigmoid 概率 | 输出形式 | 把线性分值映射成正类概率 |
| 线性边界 | 决策形状 | 当前模型在原特征空间中的基本边界假设 |
| 系数解释 | 可解释性 | 用于理解特征对正类倾向的影响方向 |
| SVC | 对比算法 | 更强调间隔与复杂边界能力 |

## 1. 为什么需要逻辑回归

对于很多分类问题，我们不只想要“分成两类”，还希望模型能告诉我们：

它认为样本属于正类的概率大概是多少？

逻辑回归给出的思路是：

- 先对特征做线性加权求和。
- 再把结果通过 Sigmoid 映射成概率。
- 最后根据概率大小做分类。

### 理解重点

- 这使逻辑回归天然兼具“分类能力”和“概率解释”。
- 当前流水线里的 `predict_proba(...)`、ROC 曲线，都是建立在这一点之上的。
- 这也是它和单纯只给硬分类边界的方法的重要区别。

## 2. 为什么当前仓库示例里它表现合理

当前逻辑回归数据来自 `make_classification(...)`，特点是：

- 二分类任务
- 高维特征
- 有效特征与冗余特征并存
- 类别具有一定分离度
- 含少量标签噪声

### 理解重点

- 这种数据并不是完美线性可分，但整体上仍适合作为线性分类模型的示例。
- 对逻辑回归来说，这是很合适的教学场景：既能展示边界，又不会因为数据过于理想而失去现实感。
- 当前数据选择和 SVC 的同心圆、Naive Bayes 的 iris 都明显不同，目标是展示线性概率分类的基本能力。

## 3. 用“线性打分 -> 概率”理解算法

可以把逻辑回归理解成：

1. 先给样本算一个线性得分。
2. 得分越大，越倾向正类；得分越小，越倾向负类。
3. 再用 Sigmoid 把这个得分变成概率。

### 示例代码

```text
先做线性组合
再做概率映射
最后根据概率输出类别
```

### 理解重点

- 这也是为什么 `coef_` 和 `intercept_` 在当前分册中值得专门观察。
- 如果把逻辑回归仅理解成“一个会分类的黑盒”，就会错过它最有价值的可解释性部分。

## 4. 为什么 `coef_` 值得重点看

### 理解重点

- `coef_` 反映特征对正类倾向的影响方向与强弱。
- 在标准化后的特征空间中，不同系数之间更容易比较。
- 当前训练日志直接打印系数和截距，就是为了让读者把模型输出和数据特征联系起来看。

## 5. 与 SVC 的直觉差异

两者的核心差异可以这样理解：

| 算法 | 更关注什么 | 核心依据 |
|---|---|---|
| LogisticRegression | 概率输出与线性边界 | 线性打分 + Sigmoid |
| SVC | 边界间隔与支持向量 | 最大间隔与判别式边界 |

### 理解重点

- 逻辑回归更强调“这个样本属于正类的概率有多大”。
- SVC 更强调“分类边界该怎么摆才最稳健”。
- 当前高维二分类数据更适合把逻辑回归作为清晰、可解释的线性基线来讲解。

## 常见坑

1. 只知道逻辑回归会分类，却说不出它为什么还能输出概率。
2. 把所有线性模型都当成同一种东西，而忽略逻辑回归的 Sigmoid 与交叉熵本质。
3. 只关注预测结果，不关注系数和截距的解释价值。
4. 忽略标准化，让系数比较与优化过程都变得不稳定。

## 小结

- 逻辑回归的直觉重点有两个：线性打分和概率映射。
- 当前仓库使用高维近线性可分数据，正好体现了它作为线性概率分类基线的优势。
- 如果已经理解“为什么它先给出线性得分，再把得分映射成概率”，就已经抓住了本分册最核心的直觉。
