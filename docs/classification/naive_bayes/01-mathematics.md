# 数学原理

> 对应代码：`model_training/classification/naive_bayes.py`、`pipelines/classification/naive_bayes.py`

> 对应对象：`GaussianNB(...)`、`model.fit(X_train, y_train)`、`model.class_prior_`

## 本章目标

1. 理解朴素贝叶斯如何用概率视角做分类。
2. 理解“朴素”条件独立假设在当前实现中意味着什么。
3. 理解为什么当前连续特征的 iris 数据自然对应 `GaussianNB`。

## 重点方法与概念速览

| 名称 | 类型 | 作用 |
|---|---|---|
| 贝叶斯公式 | 基础公式 | 从先验和似然推到后验概率 |
| 条件独立假设 | 核心假设 | 把高维联合分布拆成单特征乘积 |
| 后验概率 | 决策依据 | 用于判断样本最可能属于哪个类别 |
| 类别先验 | 概率项 | 表示各类别在训练集中的基础比例 |
| 高斯条件分布 | 概率模型 | 当前实现中对连续特征的建模方式 |
| `var_smoothing` | 超参数 | 防止方差过小导致数值不稳定 |

## 1. 朴素贝叶斯的核心思想

朴素贝叶斯不是直接拟合复杂分类边界，而是从概率角度回答一个问题：

给定当前样本特征，这个样本属于各个类别的可能性分别有多大？

### 贝叶斯公式

$$
P(Y = c_k \mid \mathbf{x}) = \frac{P(\mathbf{x} \mid Y = c_k) \, P(Y = c_k)}{P(\mathbf{x})}
$$

其中：

- $P(Y = c_k)$ 是先验概率
- $P(\mathbf{x} \mid Y = c_k)$ 是似然
- $P(Y = c_k \mid \mathbf{x})$ 是后验概率
- $P(\mathbf{x})$ 是证据项

### 理解重点

- 在分类决策时，证据项对所有类别都相同，因此通常可以忽略。
- 真正决定分类结果的，是“先验概率”和“当前特征在该类别下出现的可能性”。
- 这让 Naive Bayes 呈现出和 SVC、逻辑回归不同的生成式建模风格。

## 2. 为什么叫“朴素”

朴素贝叶斯的关键假设是：在给定类别后，各个特征之间条件独立。

$$
P(\mathbf{x} \mid Y = c_k) = P(x_1, x_2, \dots, x_d \mid Y = c_k) = \prod_{j=1}^{d} P(x_j \mid Y = c_k)
$$

### 理解重点

- 这就是“朴素”一词的来源，不是说算法粗糙，而是说它对特征关系做了强简化。
- 这个假设在真实数据里往往不严格成立，但模型依然常常能得到不错效果。
- 最大的好处是：联合分布建模难度会大幅下降，参数估计更直接。

## 3. 分类决策如何得到

根据贝叶斯公式与条件独立假设，分类规则可写为：

$$
\hat{y} = \arg\max_{c_k} P(Y = c_k) \prod_{j=1}^{d} P(x_j \mid Y = c_k)
$$

实际实现中常用对数形式避免数值下溢：

$$
\hat{y} = \arg\max_{c_k} \left[ \ln P(Y = c_k) + \sum_{j=1}^{d} \ln P(x_j \mid Y = c_k) \right]
$$

### 理解重点

- 当前模型最终输出的类别，本质上就是最大后验概率类别。
- 文档不必把重点放在复杂优化上，因为朴素贝叶斯的强项本来就不在数值求解难度，而在概率建模简洁性。

## 4. 为什么当前实现使用高斯朴素贝叶斯

当前源码使用的是：

```python
GaussianNB(var_smoothing=1e-9)
```

高斯朴素贝叶斯假设：在每个类别内，每个连续特征都服从高斯分布。

$$
P(x_j \mid Y = c_k) = \frac{1}{\sqrt{2\pi \sigma_{kj}^2}} \exp\left(-\frac{(x_j - \mu_{kj})^2}{2\sigma_{kj}^2}\right)
$$

### 理解重点

- iris 数据的特征都是连续值，因此用高斯分布建模是自然选择。
- 这也是为什么当前分册应聚焦 `GaussianNB`，而不是平均展开 `MultinomialNB`、`BernoulliNB` 等其他变体。
- 当前工程实现与理论重点必须保持一致：连续特征、多分类、GaussianNB。

## 5. `var_smoothing` 在数学上意味着什么

### 参数速览（本节）

适用参数：`var_smoothing`

| 参数名 | 当前实现取值 | 说明 |
|---|---|---|
| `var_smoothing` | `1e-9` | 向方差中加入一个很小的稳定项，避免数值问题 |

### 理解重点

- 如果某些类别下某些特征的方差非常小，计算高斯概率时容易出现数值不稳定。
- `var_smoothing` 的作用，就是给方差加一个极小平滑项，防止模型出现除零或极端概率问题。
- 当前源码把它作为唯一显式超参数，也说明这是 `GaussianNB` 最值得关注的实现细节之一。

## 6. 为什么当前 iris 数据适合这套建模方式

当前数据来自 `load_iris()`，特点是：

- 连续值特征
- 多分类任务
- 样本量不大，但结构相对清晰

### 理解重点

- 这类数据很适合用来展示“先验 + 类条件分布”的概率分类方式。
- 与文本分类常见的多项式朴素贝叶斯不同，当前分册更适合强调连续特征上的高斯假设。

## 常见坑

1. 把朴素贝叶斯理解成“简单版分类器”，而忽略它是概率模型。
2. 把“条件独立”误解成特征在原始数据中必须完全独立。
3. 不区分 GaussianNB 和其他朴素贝叶斯变体的适用数据类型。
4. 忽略 `var_smoothing` 的数值稳定作用。

## 小结

- 朴素贝叶斯的核心，是用贝叶斯公式和条件独立假设估计后验概率。
- 当前源码使用 `GaussianNB`，重点在于为每个类别上的连续特征建立高斯分布模型。
- 读懂这一层之后，再看训练、预测概率和 ROC 曲线会更顺畅。
