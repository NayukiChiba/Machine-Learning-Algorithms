# 评估与诊断

> 对应代码：`pipelines/dimensionality/pca.py`、`model_training/dimensionality/pca.py`、`result_visualization/dimensionality_plot.py`
>  
> 相关对象：`explained_variance_ratio_`、`plot_dimensionality(...)`

## 本章目标

1. 明确当前仓库实际使用了哪些评估手段，而不是泛泛讨论所有降维指标。
2. 理解解释方差比、累计解释方差和降维图分别能帮助我们判断什么。
3. 明确当前实现没有做哪些更完整的降维诊断输出，以免误读源码能力边界。

## 重点方法与概念速览

| 名称 | 类型 | 作用 |
|---|---|---|
| `explained_variance_ratio_` | 属性 | 描述每个主成分解释了多少方差 |
| 累计解释方差 | 派生量 | 描述前几个主成分总共保留了多少信息 |
| `plot_dimensionality(...)` | 函数 | 生成 2D 或 3D 降维散点图 |
| `X_transformed` / `X_3d` | 投影结果 | 当前 PCA 降维后的低维坐标 |

## 1. 当前实现真正做了什么评估

### 参数速览（本节）

适用评估手段（本节）：

1. 解释方差比
2. 累计解释方差
3. 2D / 3D 降维图

| 评估方式 | 来源 | 用途 |
|---|---|---|
| 解释方差比 | `model.explained_variance_ratio_` | 看每个主成分分别保留了多少信息 |
| 累计解释方差 | `model.explained_variance_ratio_.sum()` | 看当前主成分总共保留了多少信息 |
| 降维图 | `plot_dimensionality(...)` | 看投影后结构是否仍清晰可分 |

### 理解重点

- 当前 PCA 流水线没有显式打印重构误差、累计解释方差曲线或下游分类精度比较。
- 这并不表示这些指标不重要，而是说明本仓库当前实现更强调解释方差和可视化结构观察。
- 因此阅读这一分册时，不能把更完整的降维诊断面板想成已经在源码里实现的内容。

## 2. 解释方差比是怎么得到的

### 参数速览（本节）

适用属性：`model.explained_variance_ratio_`

| 属性名 | 当前含义 |
|---|---|
| `explained_variance_ratio_` | 每个主成分解释方差占总方差的比例 |

### 示例代码

```python
print(f"解释方差比: {model.explained_variance_ratio_.round(4)}")
print(f"累计解释方差: {model.explained_variance_ratio_.sum():.4f}")
```

### 理解重点

- 解释方差比回答的是“这个主成分到底保留了多少信息”。
- 当前 2D 模型和 3D 模型都会各自打印一组解释方差比，因此你可以直接比较不同维度下的信息保留差异。
- 这也是当前 PCA 分册最核心的定量评估线索。

## 3. 看累计解释方差时重点观察什么

### 参数速览（本节）

适用观察点（本节）：

1. 前 2 个主成分累计解释多少方差
2. 前 3 个主成分累计解释多少方差

| 现象 | 可能含义 |
|---|---|
| 前 2 个主成分已解释大量方差 | 2D 投影可能已经足够保留主要结构 |
| 前 3 个主成分明显提升累计解释方差 | 说明第三个方向仍有重要信息 |
| 累计解释方差增长较慢 | 数据信息可能更分散在更多维度上 |

### 理解重点

- 累计解释方差帮助你判断“保留这么多主成分到底够不够”。
- 当前分册专门同时训练 2D 和 3D PCA，就是为了让这种比较更直观。
- 它不是监督学习意义上的准确率，但却是降维任务里非常核心的判断依据。

## 4. 降维图是怎么生成的

### 参数速览（本节）

适用函数：`plot_dimensionality(X_transformed, y=None, explained_variance_ratio=None, title=..., dataset_name=..., model_name=..., mode='2d'/'3d')`

| 参数名 | 本例取值 | 说明 |
|---|---|---|
| `X_transformed` | 2D 或 3D 投影结果 | 当前 PCA 降维后的坐标 |
| `y` | `label` 数组 | 仅用于着色 |
| `explained_variance_ratio` | 当前模型的解释方差比 | 用于坐标轴标签标注 |
| `mode` | `'2d'` 或 `'3d'` | 选择二维或三维图 |

### 示例代码

```python
plot_dimensionality(
    X_transformed,
    y=y,
    explained_variance_ratio=model.explained_variance_ratio_,
    title="PCA 降维 (2D)",
    dataset_name=DATASET,
    model_name=MODEL,
    mode="2d",
)
```

### 理解重点

- 当前降维图并不是“模型性能图”，而是“结构可视化图”。
- 轴标签中带有 `PC1`、`PC2`、`PC3` 的解释方差比，这让图像本身就附带了信息保留程度。
- 这也是为什么当前 PCA 分册的评估主线更偏几何结构观察而不是分类指标。

## 5. 看 2D / 3D 图时重点观察什么

### 参数速览（本节）

适用观察点（本节）：

1. 结构是否压缩后仍清晰
2. 类着色后是否还能看出聚集趋势
3. 2D 与 3D 之间结构变化是否明显

| 现象 | 可能含义 |
|---|---|
| 2D 图已经结构清晰 | 前两个主成分已足够保留主信息 |
| 3D 图明显比 2D 更分开 | 第三个主成分仍然很重要 |
| 投影后结构混乱 | 当前维度压缩可能损失了较多信息 |

### 理解重点

- 当前 `label` 的作用，只是帮助你更容易观察“投影后结构是否还保持某种分组趋势”。
- 如果 2D 图已经很清楚，说明降维压缩比较成功。
- 如果 3D 图比 2D 明显更清晰，说明额外保留一个主成分确实有价值。

## 6. 当前实现没有做什么

### 参数速览（本节）

当前源码未包含的内容：

1. 重构误差打印
2. 累计解释方差曲线
3. 下游分类精度对比
4. Train/test 分离评估

| 未实现项 | 当前状态 |
|---|---|
| 重构误差 | 未在流水线中出现 |
| Scree plot / 累计解释方差曲线 | 未在流水线中出现 |
| 降维后下游任务效果对比 | 未在流水线中出现 |
| train/test split | 当前流程未使用 |

### 理解重点

- 评估章节必须以源码为准，不能把“PCA 常见诊断方法”写成“当前仓库已经实现”。
- 当前实现的评估重点是解释方差和降维图，而不是更复杂的重构或下游任务分析。
- 如果后续扩展这部分，最自然的方向是补累计解释方差曲线或重构误差输出。

## 常见坑

1. 把解释方差比误写成监督学习准确率。
2. 只看图不看解释方差比，或者只看解释方差比不看图，都容易误判降维效果。
3. 误以为当前流水线已经输出了重构误差或更多降维指标，实际源码并没有这些步骤。

## 小结

- 当前 PCA 的评估主线由两部分组成：解释方差信息和 2D / 3D 降维图。
- 它们分别从“保留了多少信息”和“保留下来的结构长什么样”两个角度解释结果。
- 只有把这两条线索一起看，才能更完整地理解当前实现的表现。
