# 思路与直觉

> 对应代码：`data_generation/dimensionality.py`、`model_training/dimensionality/lda.py`
>  
> 相关对象：`DimensionalityData.lda()`、`train_model(...)`

## 本章目标

1. 理解 LDA 为什么不是在找“方差最大”的方向，而是在找“类别最可分”的方向。
2. 理解 LDA 与 PCA 在降维目标上的根本区别。
3. 把这些直觉和当前仓库的 Wine 真实数据对应起来。

## 重点方法与概念速览

| 名称 | 类型 | 作用 |
|---|---|---|
| 类间散度 | 概念 | 衡量不同类别中心之间的分离程度 |
| 类内散度 | 概念 | 衡量同一类别内部的分散程度 |
| 判别方向 | 结果 | 让类间更远、类内更紧的投影方向 |
| `label` | 监督信息 | 当前 LDA 训练必须使用的类别标签 |

## 1. LDA 想做什么

LDA 的核心目标，不是像 PCA 那样单纯保留最大方差，而是找到一个投影方向，让不同类别投影后尽可能分开，同时同一类别内部尽可能紧凑。

### 参数速览（本节）

适用过程（本节）：

1. 观察类间距离
2. 观察类内紧凑度
3. 找到最有利于分类的投影方向

| 阶段 | 直观含义 |
|---|---|
| 拉开类别 | 不同类的中心尽量远 |
| 压紧类别 | 同一类内部尽量聚集 |
| 共同优化 | 找到最利于区分的低维表示 |

### 理解重点

- 当前 LDA 不是在做无监督压缩，而是在做“带标签信息的判别式降维”。
- 这也解释了为什么当前训练函数必须同时接收 `X_train` 和 `y_train`。
- 当前分册的阅读主轴因此应该始终围绕“可分性”展开，而不是“解释方差最大”。

## 2. 为什么当前数据特别适合讲 LDA

### 参数速览（本节）

适用数据特点（本节）：

1. 真实数据集
2. 3 个类别
3. 类间差异明显

| 数据特点 | 对 LDA 的意义 |
|---|---|
| Wine 真实数据 | 更接近实际判别任务 |
| 3 类标签 | 适合展示监督降维 |
| 类间差异明显 | 更容易看出判别方向作用 |

### 理解重点

- 如果类别本来就高度重叠，LDA 找到的投影方向也很难让图像明显分开。
- 当前 Wine 数据集类别差异较明显，因此很适合直观展示 LDA 的判别能力。
- 这也是为什么当前分册选择真实数据集，而不是手工低秩合成数据。

## 3. 为什么 `label` 在这里是真正的训练输入

### 参数速览（本节）

适用列（本节）：

1. `X`
2. `label`

| 列名 | 当前作用 |
|---|---|
| 特征矩阵 `X` | 提供化学成分特征 |
| `label` | 直接参与 LDA 训练，定义类别结构 |

### 理解重点

- 和 PCA 最大的差别就在这里：LDA 真的会用到类别标签。
- 没有标签，就无法定义类间散度和类内散度，也就无法做当前这套判别式降维。
- 这点必须在文档里反复强调，否则最容易把 LDA 和 PCA 写混。

## 4. 为什么当前只输出 2D LDA 图

### 参数速览（本节）

适用事实（本节）：

1. 当前类别数 `K = 3`
2. LDA 最大判别维度 `K - 1 = 2`

| 项目 | 当前含义 |
|---|---|
| 类别数 | 3 |
| 最大可降维数 | 2 |

### 理解重点

- 当前实现只训练 2D LDA，不是因为工程上随意选了 2，而是因为 3 类数据本来就最多只能提取 2 个判别方向。
- 这和 PCA 可以继续训练 3D 模型的逻辑完全不同。
- 当前分册的 2D 输出因此既是工程实现选择，也是数学上限约束的直接体现。

## 5. 与 PCA 相比，LDA 的优势和边界

### 参数速览（本节）

适用对比对象（本节）：

1. PCA
2. LDA

| 维度 | PCA | LDA |
|---|---|---|
| 学习方式 | 无监督 | 有监督 |
| 目标 | 最大投影方差 | 最大类间 / 类内散度比 |
| 标签是否参与 | 否 | 是 |
| 降维上限 | 可由数据维度决定 | `K - 1` |

### 理解重点

- PCA 更适合做“结构压缩”和无监督可视化。
- LDA 更适合做“判别压缩”，即让类别在低维空间里更容易分开。
- 当前分册因此要强调它不是“另一个 PCA”，而是一种目标不同的监督降维方法。

## 6. 直觉如何映射到当前训练日志和输出

### 参数速览（本节）

适用输出项（分项）：

1. `n_components`
2. 可选 `explained_variance_ratio_`
3. 2D LDA 图

| 输出项 | 可以帮助判断什么 |
|---|---|
| `n_components` | 当前保留了多少个判别方向 |
| `explained_variance_ratio_` | 若存在，可看各判别方向贡献 |
| 2D 图 | 类别在判别空间中是否被拉开 |

### 理解重点

- 当前 LDA 分册最重要的结果，不是监督精度数字，而是判别方向投影后的可分性结构。
- 控制台中的解释方差信息若存在，是辅助理解，不应取代对图像结构的观察。
- 这说明 LDA 分册的核心观察点是“类别是否在低维空间中更可分”。

## 常见坑

1. 把 LDA 当成无监督降维，忽略它必须依赖标签。
2. 把 LDA 与 PCA 只看成“两个降维算法”，却忽略它们优化目标根本不同。
3. 忽略 `K-1` 维上限，误以为 LDA 可以像 PCA 一样随意继续增加输出维度。

## 小结

- LDA 的核心直觉，是在低维空间里尽量拉开不同类别、压紧同类样本。
- 当前 Wine 数据集非常适合展示这种有监督判别式降维效果。
- 把这些直觉建立起来之后，再看数学原理、训练流程和可视化结果会更容易。
