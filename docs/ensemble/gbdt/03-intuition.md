# 思路与直觉

> 对应代码：`data_generation/ensemble.py`、`model_training/ensemble/gbdt.py`
>  
> 相关对象：`EnsembleData.gbdt()`、`train_model(...)`

## 本章目标

1. 理解 GBDT 为什么不是并行平均很多树，而是逐轮串行纠错。
2. 理解“负梯度作为伪残差”为什么是 GBDT 的核心直觉。
3. 把这些直觉和当前仓库的中等难度多分类数据对应起来。

## 重点方法与概念速览

| 名称 | 类型 | 作用 |
|---|---|---|
| Boosting | 集成思想 | 逐轮叠加弱学习器修正当前错误 |
| 伪残差 | 优化对象 | 当前轮需要重点拟合的错误方向 |
| 学习率 | 缩减机制 | 控制每轮更新步长 |
| 子采样 | 随机化机制 | 控制训练稳定性与泛化 |

## 1. GBDT 想做什么

GBDT 的核心目标，不是训练很多棵互不相关的树再取平均，而是训练一串有顺序依赖的树，让每一轮新树去修正当前模型还没学好的部分。

### 参数速览（本节）

适用过程（本节）：

1. 当前模型给出预测
2. 计算当前错误方向
3. 新树去拟合这部分错误
4. 多轮叠加形成最终模型

| 阶段 | 直观含义 |
|---|---|
| 初始阶段 | 先有一个基础模型 |
| 新树加入 | 专门去补当前模型还没解释好的部分 |
| 多轮叠加 | 最终形成强的加法模型 |

### 理解重点

- Bagging 更像“并行投票或平均”。
- GBDT 更像“连续纠错”。
- 当前源码中的 `n_estimators=200`，就代表这种纠错轮数规模。

## 2. 为什么“伪残差”是最关键的直觉

### 参数速览（本节）

适用概念：负梯度 / 伪残差

| 概念 | 直观含义 |
|---|---|
| 真实残差 | 预测和真实之间的直接误差 |
| 伪残差 | 损失函数对当前预测的负梯度 |

### 理解重点

- 在平方损失下，负梯度就等于普通残差，所以这个直觉最容易理解。
- 在更一般的损失函数下，GBDT 实际拟合的是“当前最该往哪个方向修正”的信号，而不一定是字面残差。
- 这就是为什么 GBDT 能推广到分类问题，而不只是回归问题。

## 3. 为什么 GBDT 比单棵树更强

### 参数速览（本节）

适用能力（本节）：

1. 边界逐步修正
2. 多轮组合表达复杂结构
3. 降低偏差

| 能力 | 直观解释 |
|---|---|
| 边界修正 | 每一轮都针对当前错误做补偿 |
| 复杂表达 | 多棵树组合后能拟合更复杂边界 |
| 降低偏差 | 相比单棵浅树，整体模型更强 |

### 理解重点

- 单棵树的分裂一旦确定，表达力就受限于那一棵树本身。
- GBDT 通过不断加树，可以逐步修正原来分错的区域。
- 这也是为什么它通常比单棵树分类更强，但训练和调参也更复杂。

## 4. 为什么当前数据特别适合讲 GBDT

### 参数速览（本节）

适用数据特点（本节）：

1. 3 分类
2. 中等难度
3. 类间间隔适中
4. 有效特征与冗余特征并存

| 数据特点 | 对 GBDT 的意义 |
|---|---|
| 3 分类 | 适合展示多分类边界修正 |
| 类间间隔适中 | 单棵树不一定足够，boosting 更有发挥空间 |
| 冗余特征存在 | 有助于观察特征重要性与模型筛选能力 |

### 理解重点

- 如果分类边界非常简单，单棵树就可能已经足够。
- 当前数据故意设计成“中等难度”，是为了让 GBDT 的逐轮纠错优势更明显。
- 这也是为什么当前流水线既输出混淆矩阵，也输出学习曲线和特征重要性图。

## 5. 与 Bagging、单棵树和 XGBoost 相比，GBDT 的优势和边界

### 参数速览（本节）

适用对比对象（本节）：

1. 单棵决策树
2. Bagging
3. GBDT
4. XGBoost

| 维度 | 单棵树 | Bagging | GBDT | XGBoost |
|---|---|---|---|---|
| 集成方式 | 无 | 并行平均 | 串行纠错 | 串行纠错 + 更强优化 |
| 主要目标 | 学一棵树 | 降低方差 | 降低偏差 | 更稳更强的 boosting |
| 数学主线 | 树分裂 | 重采样平均 | 负梯度拟合 | 二阶优化 + 正则化 |

### 理解重点

- Bagging 和 GBDT 的最大差别，在于一个强调并行平均，一个强调串行修正。
- XGBoost 可以看成 GBDT 的更工程化、更强化版本，但 GBDT 仍然是理解 boosting 思想的最好起点之一。
- 当前分册因此更强调“伪残差”而不是二阶优化和复杂正则项。

## 6. 直觉如何映射到当前训练日志和输出

### 参数速览（本节）

适用输出项（分项）：

1. `n_estimators`
2. `learning_rate`
3. `max_depth`
4. `subsample`

| 日志项 | 可以帮助判断什么 |
|---|---|
| `n_estimators` | 串行纠错轮数 |
| `learning_rate` | 每轮修正步长 |
| `max_depth` | 基学习器复杂度 |
| `subsample` | 是否引入随机梯度提升风格采样 |

### 理解重点

- 当前训练日志最关键的信息，不是线性系数，也不是单棵树深度，而是一组控制 boosting 行为的超参数。
- 这些参数共同决定模型是更激进还是更保守、更强拟合还是更稳定。
- 这也说明 GBDT 分册的核心观察点，是“伪残差驱动的串行学习过程如何被超参数约束”。

## 常见坑

1. 把 GBDT 误解成“很多树投票”，忽略它其实是串行拟合错误方向。
2. 只改 `n_estimators`，忽略 `learning_rate` 和 `max_depth` 也会显著影响结果。
3. 把 GBDT 和 XGBoost 混为一谈，直接把二阶优化和强正则化当成当前分册的主轴。

## 小结

- GBDT 的核心直觉，是用一串树去逐轮修正当前模型的错误方向。
- 负梯度作为伪残差，是理解它为什么既能做回归也能做分类的关键。\n+- 当前中等难度多分类数据正是为了把这种“串行纠错、逐步修正边界”的特性展示出来。
