# 评估与诊断

> 对应代码：`pipelines/ensemble/xgboost.py`、`result_visualization/residual_plot.py`、`result_visualization/feature_importance.py`
>  
> 相关对象：`plot_residuals(...)`、`plot_feature_importance(...)`

## 本章目标

1. 明确当前仓库实际使用了哪些评估手段，而不是泛泛讨论所有回归指标。
2. 理解残差图和特征重要性图分别能帮助我们诊断什么。
3. 明确当前实现没有做哪些数值指标和可视化输出，以免误读源码能力边界。

## 重点方法与概念速览

| 名称 | 类型 | 作用 |
|---|---|---|
| `plot_residuals(...)` | 函数 | 生成预测-真实图和残差分布图 |
| `plot_feature_importance(...)` | 函数 | 生成特征重要性柱状图 |
| `feature_importances_` | 属性 | 树模型中各特征的重要性来源 |
| `y_pred` | 预测结果 | 当前 XGBoost 对测试集的回归输出 |

## 1. 当前实现真正做了什么评估

### 参数速览（本节）

适用评估手段（本节）：

1. 残差图
2. 特征重要性图

| 评估方式 | 来源 | 用途 |
|---|---|---|
| 残差图 | `plot_residuals(...)` | 观察测试集上的整体拟合情况与误差分布 |
| 特征重要性图 | `plot_feature_importance(...)` | 观察哪些特征对 boosting 模型更重要 |

### 理解重点

- 当前 XGBoost 流水线没有显式打印 `MSE`、`MAE`、`RMSE`、`R^2` 等数值指标。
- 这并不表示这些指标不重要，而是说明本仓库当前实现更强调图像化诊断和特征层面的观察。
- 因此阅读这一分册时，不能把“指标表格”想成已经在源码里实现的内容。

## 2. 残差图是怎么生成的

### 参数速览（本节）

适用函数：`plot_residuals(y_true, y_pred, title='残差分析', dataset_name='default', model_name='model', figsize=(14, 5))`

| 参数名 | 本例取值 | 说明 |
|---|---|---|
| `y_true` | `y_test` | 测试集真实值 |
| `y_pred` | 模型预测值 | 测试集预测结果 |
| `dataset_name` | `"xgboost"` | 输出目录名 |
| `model_name` | `"xgboost"` | 输出文件名前缀 |
| `figsize` | `(14, 5)` | 图像尺寸 |

### 示例代码

```python
plot_residuals(
    y_test,
    y_pred,
    title="XGBoost 残差分析",
    dataset_name=DATASET,
    model_name=MODEL,
)
```

### 理解重点

- 第一张子图看“预测值是否靠近真实值对角线”，适合粗看拟合效果。
- 第二张子图看“残差是否围绕 0 随机分布”，适合发现系统偏差、异常点和局部拟合不足。
- 对当前真实房价数据来说，这一步是最直接的误差分布诊断方式。

## 3. 特征重要性图是怎么生成的

### 参数速览（本节）

适用函数：`plot_feature_importance(model, feature_names=None, top_n=None, title='特征重要性', dataset_name='default', model_name='model', figsize=(10, 7))`

| 参数名 | 本例取值 | 说明 |
|---|---|---|
| `model` | 训练好的 XGBoost 模型 | 提供特征重要性来源 |
| `feature_names` | `list(X.columns)` | 为每个重要性值提供真实列名 |
| `top_n` | `None` | 当前实现展示全部特征 |
| `dataset_name` | `"xgboost"` | 输出目录名 |
| `model_name` | `"xgboost"` | 输出文件名前缀 |

### 示例代码

```python
plot_feature_importance(
    model,
    feature_names=feature_names,
    title="XGBoost 特征重要性",
    dataset_name=DATASET,
    model_name=MODEL,
)
```

### 理解重点

- 对当前 XGBoost 分册来说，特征重要性图是理解模型行为的关键补充信息。
- 它反映的是特征对树分裂和整体 boosting 模型贡献的相对大小，而不是线性模型里的系数。
- 因此“重要性高”不等于“这个特征和房价线性正相关”，两者含义不同。

## 4. 看残差图时重点观察什么

### 参数速览（本节）

适用观察点（本节）：

1. 对角线贴合程度
2. 残差围绕 0 的分布
3. 离群点与局部结构

| 现象 | 可能含义 |
|---|---|
| 点大多贴近 `y=x` | 整体预测较稳定 |
| 残差围绕 0 随机散开 | 没有明显系统偏差 |
| 残差整体偏正或偏负 | 模型存在系统性高估或低估 |
| 某些区间误差明显集中 | 模型对局部区域拟合可能不足 |
| 少量极端散点 | 可能有异常样本或局部难例 |

### 理解重点

- XGBoost 虽然表达力很强，但并不意味着残差一定理想。
- 如果某些局部区域始终预测不好，残差图仍会暴露出明显结构。
- 这也是为什么本分册不能只看特征重要性，还要看预测误差分布。

## 5. 看特征重要性图时重点观察什么

### 参数速览（本节）

适用观察点（本节）：

1. 重要性是否集中在少数特征
2. 地理位置和收入类特征是否占主导
3. 重要性分布是否合理

| 现象 | 可能含义 |
|---|---|
| 少数特征重要性非常高 | 模型主要依赖这些特征做分裂 |
| 多个特征重要性接近 | 模型在组合利用多种信息 |
| 重要性分布异常集中 | 可能需要结合残差图检查是否过拟合或结构偏置 |

### 理解重点

- 特征重要性图帮助你理解“模型主要看什么”，但不能替代误差评估。
- 残差图帮助你理解“模型最终预测得怎么样”。
- 只有把“重要性 + 残差图”两条线索结合起来，才能更完整地读懂当前 XGBoost 模型。

## 6. 当前实现没有做什么

### 参数速览（本节）

当前源码未包含的内容：

1. 显式数值指标打印
2. 学习曲线
3. 交叉验证
4. 早停训练

| 未实现项 | 当前状态 |
|---|---|
| `MSE` / `MAE` / `RMSE` / `R^2` 打印 | 未在流水线中出现 |
| 学习曲线 | 当前流程未调用相关函数 |
| 交叉验证 | 未在流水线中出现 |
| `early_stopping_rounds` 等早停逻辑 | 当前训练封装未使用 |

### 理解重点

- 评估章节必须以源码为准，不能把“XGBoost 常见训练技巧”写成“当前仓库已经实现”。
- 当前实现的评估重点是残差图和特征重要性图，而不是数值指标面板或早停训练记录。
- 如果后续扩展这部分，最自然的方向是补指标打印、学习曲线或验证集早停流程。

## 常见坑

1. 只看特征重要性图，不看残差图，误把“重要性高”当成“预测一定好”。
2. 只看残差图，不结合特征重要性，错过对模型关注重点的观察。
3. 误以为当前流水线已经输出了完整的 `R^2`、`MSE` 指标表或学习曲线，实际源码并没有这些步骤。

## 小结

- 当前 XGBoost 的评估主线由两部分组成：残差图和特征重要性图。
- 残差图负责看误差分布，特征重要性图负责看模型主要依赖哪些特征。
- 只有把这两条线索一起看，才能更完整地理解当前实现的表现。
