# Pandas 数据导入与导出

> 对应脚本：`Basic/Pandas/02_io.py`

## 1. CSV 文件读取

### 方法重点

- `read_csv` 是 Pandas 最常用的 IO 函数，覆盖大部分表格数据场景。
- 支持指定分隔符、编码、跳行、列选择等丰富参数。
- `to_csv` 默认会写入索引列，通常需要 `index=False`。

### 参数速览（本节）

1. `pd.read_csv(filepath_or_buffer, sep=',', header='infer', names=None, usecols=None, nrows=None, skiprows=None, encoding=None, ...)`

| 参数名 | 本例取值 | 说明 |
|---|---|---|
| `filepath_or_buffer` | `"temp_demo.csv"` | 文件路径或 URL |
| `sep` | `','`（默认） | 字段分隔符 |
| `header` | `'infer'`（默认） | 表头行号，`None` 表示无表头 |
| `names` | `None`（默认） | 自定义列名列表 |
| `usecols` | `None`（默认） | 只读取指定列 |
| `nrows` | `None`（默认） | 只读取前 n 行 |
| `skiprows` | `None`（默认） | 跳过指定行 |
| `encoding` | `None`（默认） | 文件编码，常用 `'utf-8'`、`'gbk'` |

2. `df.to_csv(path_or_buf, sep=',', index=True, encoding=None, ...)`

| 参数名 | 本例取值 | 说明 |
|---|---|---|
| `path_or_buf` | `"temp_demo.csv"` | 输出文件路径 |
| `index` | `False` | 不写入行索引 |

### 示例代码

```python
import pandas as pd
import os

# 创建示例数据并保存
df = pd.DataFrame({
    "Name": ["Alice", "Bob", "Charlie"],
    "Age": [25, 30, 35],
    "Score": [85.5, 90.0, 78.5],
})

temp_file = "temp_demo.csv"
df.to_csv(temp_file, index=False)

# 读取 CSV
df_read = pd.read_csv(temp_file)
print(df_read)

os.remove(temp_file)
```

### 结果输出

```text
      Name  Age  Score
0    Alice   25   85.5
1      Bob   30   90.0
2  Charlie   35   78.5
```

### 理解重点

- `read_csv` 默认第一行为表头（`header='infer'`）。
- 处理中文 CSV 时常需 `encoding='utf-8'` 或 `encoding='gbk'`。
- `usecols` 和 `nrows` 可用于大文件的快速预览。

## 2. Excel 文件读写

### 方法重点

- Excel 读写依赖第三方库：读取需要 `openpyxl`（`.xlsx`）或 `xlrd`（`.xls`），写入需要 `openpyxl`。
- `sheet_name` 支持工作表名称或索引号。
- 与 CSV 不同，Excel 文件天然支持多工作表。

### 参数速览（本节）

1. `pd.read_excel(io, sheet_name=0, header=0, usecols=None, engine=None, ...)`

| 参数名 | 典型取值 | 说明 |
|---|---|---|
| `io` | `"file.xlsx"` | Excel 文件路径 |
| `sheet_name` | `0`、`"Sheet1"` | 工作表名或索引号 |
| `header` | `0`（默认） | 表头行号 |
| `usecols` | `"A:C"` 或 `[0,1,2]` | 读取的列范围 |
| `engine` | `"openpyxl"`、`"xlrd"` | 解析引擎 |

2. `df.to_excel(excel_writer, sheet_name='Sheet1', index=True, ...)`

| 参数名 | 典型取值 | 说明 |
|---|---|---|
| `excel_writer` | `"output.xlsx"` | 输出文件路径 |
| `sheet_name` | `"Sheet1"` | 工作表名 |
| `index` | `False` | 不写入行索引 |

### 示例代码（概念演示）

```python
import pandas as pd

# 读取 Excel
# df = pd.read_excel("file.xlsx", sheet_name="Sheet1")

# 保存到 Excel
# df.to_excel("output.xlsx", sheet_name="Sheet1", index=False)
```

### 理解重点

- 安装依赖：`pip install openpyxl`（`.xlsx`）或 `pip install xlrd`（旧版 `.xls`）。
- `sheet_name=None` 可以一次读取所有工作表，返回字典 `{sheet_name: DataFrame}`。
- 处理大型 Excel 文件时，`usecols` 可以显著减少内存占用。

## 3. JSON 文件读写

### 方法重点

- JSON 是 Web 数据交换的标准格式，Pandas 提供原生支持。
- `orient` 参数决定 JSON 的结构化方式，影响序列化和反序列化行为。
- `force_ascii=False` 保证中文不被转义。

### 参数速览（本节）

1. `df.to_json(path_or_buf, orient='columns', indent=None, force_ascii=True, ...)`

| 参数名 | 本例取值 | 说明 |
|---|---|---|
| `path_or_buf` | `"temp_demo.json"` | 输出文件路径 |
| `orient` | `"records"` | JSON 结构方式 |
| `indent` | `2` | 缩进空格数 |
| `force_ascii` | `False` | 不转义非 ASCII 字符 |

2. `pd.read_json(path_or_buf, orient=None, ...)`

| 参数名 | 本例取值 | 说明 |
|---|---|---|
| `path_or_buf` | `"temp_demo.json"` | JSON 文件路径 |

### 示例代码

```python
import pandas as pd
import os

df = pd.DataFrame({
    "Name": ["Alice", "Bob"],
    "Age": [25, 30],
    "Skills": [["Python", "SQL"], ["Java", "C++"]],
})

# 保存为 JSON
temp_file = "temp_demo.json"
df.to_json(temp_file, orient="records", indent=2, force_ascii=False)

# 读取 JSON
df_read = pd.read_json(temp_file)
print(df_read)

os.remove(temp_file)
```

### 结果输出

```text
    Name  Age        Skills
0  Alice   25  [Python, SQL]
1    Bob   30    [Java, C++]
```

### 理解重点

- `orient` 参数选项：
  - `'records'`：`[列:值}, {列:值}]` — 最常用，与 API 数据格式一致。
  - `'columns'`：`{列: {索引:值}}` — 默认格式。
  - `'index'`：`{索引: {列:值}}` — 以行为单位。
  - `'values'`：`[[值, 值], [值, 值]]` — 纯数组。
- 包含嵌套结构（如列表、字典）的列可以正常序列化为 JSON。

## 4. 数据导出方法总览

### 方法重点

- Pandas 支持多种导出格式，覆盖文件、数据库、Web 等场景。
- `to_string()` 适合在终端或日志中输出格式化表格。
- 二进制格式（Pickle、Parquet）读写速度和压缩率远优于文本格式。

### 参数速览（本节）

| 导出方法 | 输出格式 | 说明 |
|---|---|---|
| `df.to_csv(...)` | CSV 文本 | 最通用的文本格式 |
| `df.to_excel(...)` | Excel | 需要 openpyxl |
| `df.to_json(...)` | JSON | Web 数据交换 |
| `df.to_html(...)` | HTML 表格 | 嵌入网页 |
| `df.to_sql(...)` | SQL 数据库 | 需要 SQLAlchemy 连接 |
| `df.to_pickle(...)` | Pickle 序列化 | Python 原生，速度快 |
| `df.to_parquet(...)` | Parquet 列存 | 大数据场景首选 |
| `df.to_string()` | 格式化字符串 | 终端/日志输出 |

### 示例代码

```python
import pandas as pd

df = pd.DataFrame({"A": [1, 2, 3], "B": ["x", "y", "z"]})

# to_string 输出
print(df.to_string())
```

### 结果输出

```text
   A  B
0  1  x
1  2  y
2  3  z
```

### 理解重点

- 文本格式（CSV、JSON）通用性强但效率低，二进制格式（Parquet、Pickle）效率高但可移植性弱。
- `to_parquet` 配合 `pyarrow` 或 `fastparquet` 引擎，适合大规模数据存储。
- `to_sql` 需要 SQLAlchemy 连接对象，适合将结果直接写入数据库。

## 常见坑

1. `to_csv` 默认 `index=True`，读回来会多一列无名索引，建议导出时加 `index=False`。
2. `read_csv` 遇到中文乱码时，先尝试 `encoding='utf-8'`，再尝试 `encoding='gbk'`。
3. `read_excel` 需要额外安装引擎库，否则报 `ImportError`。
4. JSON 的 `orient` 不匹配时，`read_json` 解析结果会错位。

## 小结

- IO 是数据分析的起点和终点，Pandas 提供了覆盖面极广的读写接口。
- CSV 是最通用的格式，JSON 适合 Web 场景，Parquet 适合大数据场景。
- 理解每种格式的参数特性，可以避免大量数据导入导出问题。
