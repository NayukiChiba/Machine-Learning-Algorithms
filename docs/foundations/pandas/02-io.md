# Pandas 数据导入与导出

## 本章目标

1. 掌握 CSV 文件的读取 / 写入与常用参数。
2. 了解 Excel 文件读写的依赖与关键参数。
3. 学会 JSON 文件读写及 `orient` 的选择。
4. 熟悉 Pandas 支持的其他导出格式（HTML、SQL、Pickle、Parquet）。

## 重点方法与概念速览

| 名称 | 类型 | 作用 |
|---|---|---|
| `pd.read_csv(...)` | 函数 | 从 CSV 读取数据 |
| `df.to_csv(...)` | 方法 | 导出为 CSV |
| `pd.read_excel(...)` | 函数 | 从 Excel 读取数据 |
| `df.to_excel(...)` | 方法 | 导出为 Excel |
| `pd.read_json(...)` | 函数 | 从 JSON 读取数据 |
| `df.to_json(...)` | 方法 | 导出为 JSON |
| `df.to_html(...)` | 方法 | 导出为 HTML 表格 |
| `df.to_sql(...)` | 方法 | 写入 SQL 数据库 |
| `df.to_pickle(...)` | 方法 | Python 原生序列化 |
| `df.to_parquet(...)` | 方法 | 列存二进制格式 |
| `df.to_string(...)` | 方法 | 转为格式化字符串 |

## CSV 文件

### `pd.read_csv`

#### 作用

从 CSV（或兼容的分隔文本）读取数据到 `DataFrame`。是 Pandas 最常用的 IO 函数，参数极多，覆盖绝大多数文本表格场景。

#### 重点方法

```python
pd.read_csv(filepath_or_buffer, sep=',', delimiter=None, header='infer',
            names=None, index_col=None, usecols=None, dtype=None,
            engine=None, converters=None, skiprows=None, skipfooter=0,
            nrows=None, na_values=None, keep_default_na=True, na_filter=True,
            parse_dates=False, date_format=None, dayfirst=False,
            encoding=None, chunksize=None, compression='infer')
```

#### 参数

| 参数名                | 本例取值              | 说明                                                                     |
| --------------------- | --------------------- | ------------------------------------------------------------------------ |
| `filepath_or_buffer`  | `"temp_demo.csv"`     | 文件路径、URL 或类文件对象                                               |
| `sep` / `delimiter`   | `','`（默认）         | 字段分隔符；`'\t'` 读 TSV                                                |
| `header`              | `'infer'`（默认）、`None` | 表头行号；`None` 表示没有表头                                       |
| `names`               | `None`（默认）        | 自定义列名列表；与 `header=None` 搭配最常用                              |
| `index_col`           | `None`（默认）、`0`、`'id'` | 将哪一列作为行索引                                                 |
| `usecols`             | `None`（默认）、`['A', 'B']` | 只读取指定列，减少内存占用                                      |
| `dtype`               | `None`（默认）、`{'age': int}` | 指定列的 dtype                                                 |
| `engine`              | `None`、`'c'`、`'python'` | 解析引擎；`'c'` 更快但功能少                                         |
| `skiprows`            | `None`（默认）、`1`、`[0, 2]` | 跳过开头多少行或指定行                                         |
| `nrows`               | `None`（默认）、`1000` | 只读取前 n 行，用于大文件预览                                           |
| `na_values`           | `None`（默认）、`['N/A', 'NaN', '?']` | 额外的缺失值标识                                        |
| `parse_dates`         | `False`（默认）、`['date']` | 解析为日期的列名列表                                              |
| `encoding`            | `None`（默认）、`'utf-8'`、`'gbk'` | 文件编码                                                   |
| `chunksize`           | `None`（默认）、`10000` | 分块读取，返回迭代器；大文件必用                                       |
| `compression`         | `'infer'`（默认）、`'gzip'` | 自动识别 `.gz` / `.bz2` / `.zip`；可强制指定                        |

### `df.to_csv`

#### 作用

将 `DataFrame` 写入 CSV 文件或字符串。

#### 重点方法

```python
df.to_csv(path_or_buf=None, sep=',', na_rep='', float_format=None,
          columns=None, header=True, index=True, index_label=None,
          mode='w', encoding=None, compression='infer',
          date_format=None, lineterminator=None)
```

#### 参数

| 参数名           | 本例取值                  | 说明                                                   |
| ---------------- | ------------------------- | ------------------------------------------------------ |
| `path_or_buf`    | `"temp_demo.csv"`、`None` | 输出路径；`None` 则返回字符串                          |
| `sep`            | `','`（默认）             | 分隔符                                                 |
| `na_rep`         | `''`（默认）、`'NaN'`     | 缺失值的文本表示                                       |
| `float_format`   | `None`（默认）、`'%.4f'`  | 浮点数格式化                                           |
| `columns`        | `None`（默认）            | 只导出指定列                                           |
| `header`         | `True`（默认）            | 是否写表头                                             |
| `index`          | `True`（默认）、`False`   | 是否写行索引（绝大多数场景设 `False`）                 |
| `mode`           | `'w'`（默认）、`'a'`      | 写入模式：`'w'` 覆盖、`'a'` 追加                       |
| `encoding`       | `None`（默认）、`'utf-8'` | 文件编码                                               |
| `compression`    | `'infer'`（默认）、`'gzip'` | 压缩格式，`.gz` / `.zip` 自动推断                    |
| `lineterminator` | `None`（默认）            | 行结束符，跨平台时用于统一换行                         |

### 综合示例

#### 示例代码

```python
import pandas as pd
import os

df = pd.DataFrame({
    "Name": ["Alice", "Bob", "Charlie"],
    "Age": [25, 30, 35],
    "Score": [85.5, 90.0, 78.5],
})

df.to_csv("temp_demo.csv", index=False)

df_read = pd.read_csv("temp_demo.csv")
print(df_read)

os.remove("temp_demo.csv")
```

#### 输出

```text
      Name  Age  Score
0    Alice   25   85.5
1      Bob   30   90.0
2  Charlie   35   78.5
```

#### 理解重点

- **`index=False` 几乎是必加**：否则 `to_csv` 会写入一列无名索引，`read_csv` 再读时会多出一列。
- 中文乱码常见解法：先试 `encoding='utf-8-sig'`，再试 `encoding='gbk'`。
- 大文件用 `chunksize=10000` 分块读取，避免一次性占满内存。

## Excel 文件

### `pd.read_excel`

#### 作用

读取 Excel 工作表。需要额外安装 `openpyxl`（`.xlsx`）或 `xlrd`（旧版 `.xls`）。

#### 重点方法

```python
pd.read_excel(io, sheet_name=0, header=0, names=None, index_col=None,
              usecols=None, dtype=None, engine=None, skiprows=None,
              nrows=None, na_values=None, parse_dates=False)
```

#### 参数

| 参数名       | 本例取值                           | 说明                                                              |
| ------------ | ---------------------------------- | ----------------------------------------------------------------- |
| `io`         | `"file.xlsx"`                      | Excel 文件路径或 `ExcelFile` 对象                                 |
| `sheet_name` | `0`（默认）、`"Sheet1"`、`None`、`[0, 1]` | 工作表；`None` 返回所有表的字典；列表返回指定多表的字典        |
| `header`     | `0`（默认）、`None`                | 表头所在行号                                                      |
| `usecols`    | `None`（默认）、`"A:C"`、`[0,1,2]`、`"A,C,E"` | 读取的列                                                 |
| `dtype`      | `None`（默认）                     | 指定 dtype                                                        |
| `engine`     | `None`、`"openpyxl"`、`"xlrd"`     | 解析引擎；现代 `.xlsx` 用 `openpyxl`                              |
| `skiprows`   | `None`（默认）、`1`                | 跳过开头行数                                                      |

### `df.to_excel`

#### 作用

将 `DataFrame` 写入 Excel 工作表。

#### 重点方法

```python
df.to_excel(excel_writer, sheet_name='Sheet1', na_rep='', float_format=None,
            columns=None, header=True, index=True, index_label=None,
            startrow=0, startcol=0, engine=None, merge_cells=True,
            freeze_panes=None)
```

#### 参数

| 参数名         | 本例取值               | 说明                                                               |
| -------------- | ---------------------- | ------------------------------------------------------------------ |
| `excel_writer` | `"output.xlsx"`、`ExcelWriter` | 路径或 `ExcelWriter` 对象（多表写入时用）                    |
| `sheet_name`   | `'Sheet1'`（默认）     | 工作表名                                                           |
| `na_rep`       | `''`（默认）           | 缺失值文本                                                         |
| `columns`      | `None`（默认）         | 只导出指定列                                                       |
| `header`       | `True`（默认）         | 是否写表头                                                         |
| `index`        | `True`（默认）、`False`| 是否写行索引                                                       |
| `startrow`     | `0`（默认）            | 起始行（留表格外空间）                                             |
| `startcol`     | `0`（默认）            | 起始列                                                             |
| `engine`       | `None`、`'openpyxl'`   | 写入引擎                                                           |
| `freeze_panes` | `None`（默认）、`(1,0)`| 冻结窗口；`(1, 0)` 冻结表头                                        |

### 示例代码

```python
import pandas as pd

# 多表写入 Excel
with pd.ExcelWriter("output.xlsx", engine="openpyxl") as writer:
    df.to_excel(writer, sheet_name="Data", index=False)
    summary.to_excel(writer, sheet_name="Summary", index=False)

# 一次读取所有表
all_sheets = pd.read_excel("output.xlsx", sheet_name=None)
print(list(all_sheets.keys()))
```

#### 理解重点

- `pip install openpyxl` 是 `.xlsx` 读写的必备依赖。
- `sheet_name=None` 返回字典 `{sheet_name: DataFrame}`，批量处理多表场景。
- 处理大型 Excel 时用 `usecols` 限定列，显著降低内存。

## JSON 文件

### `pd.read_json`

#### 作用

从 JSON 文件 / 字符串读取数据到 `DataFrame`。

#### 重点方法

```python
pd.read_json(path_or_buf, orient=None, typ='frame', dtype=None,
             convert_dates=True, encoding='utf-8', lines=False,
             chunksize=None, compression='infer')
```

#### 参数

| 参数名         | 本例取值              | 说明                                                                    |
| -------------- | --------------------- | ----------------------------------------------------------------------- |
| `path_or_buf`  | `"temp_demo.json"`    | 文件路径、URL 或 JSON 字符串                                            |
| `orient`       | `None`（默认按 `typ` 推断）、`'records'`、`'columns'`、`'index'`、`'values'`、`'split'`、`'table'` | JSON 结构模式（见下表） |
| `typ`          | `'frame'`（默认）、`'series'` | 返回类型                                                         |
| `dtype`        | `None`（默认）        | 显式指定列 dtype                                                        |
| `convert_dates`| `True`（默认）        | 尝试自动解析日期列                                                      |
| `encoding`     | `'utf-8'`（默认）     | 文件编码                                                                |
| `lines`        | `False`（默认）       | `True` 时按 ndjson / jsonlines 格式读取                                 |
| `chunksize`    | `None`（默认）        | 仅 `lines=True` 时可用；分块读取                                        |

### `df.to_json`

#### 作用

将 `DataFrame` 写为 JSON。

#### 重点方法

```python
df.to_json(path_or_buf=None, orient=None, date_format=None, double_precision=10,
           force_ascii=True, date_unit='ms', default_handler=None, lines=False,
           compression='infer', index=None, indent=None)
```

#### 参数

| 参数名             | 本例取值                 | 说明                                                      |
| ------------------ | ------------------------ | --------------------------------------------------------- |
| `path_or_buf`      | `"temp_demo.json"`、`None` | 输出路径；`None` 则返回字符串                          |
| `orient`           | `None`、`'records'`、`'columns'`、`'index'`、`'values'`、`'split'`、`'table'` | JSON 结构模式 |
| `date_format`      | `None`、`'epoch'`、`'iso'` | 日期格式                                                |
| `double_precision` | `10`（默认）             | 浮点数精度                                                |
| `force_ascii`      | `True`（默认）、`False`  | 是否转义非 ASCII 字符；含中文**必设 `False`**             |
| `lines`            | `False`（默认）          | `True` 且 `orient='records'` 时输出 ndjson 格式           |
| `indent`           | `None`（默认）、`2`      | 缩进空格数，美化输出                                      |

### `orient` 的五种结构

| `orient`    | 结构样式                                    | 典型用途                    |
| ----------- | ------------------------------------------- | --------------------------- |
| `'records'` | `[{col: val}, {col: val}]`                  | API / 日志行：**最常用**    |
| `'columns'` | `{col: {index: val}}`                       | 默认                        |
| `'index'`   | `{index: {col: val}}`                       | 按行序列化                  |
| `'values'`  | `[[val, val], [val, val]]`                  | 纯数组                      |
| `'split'`   | `{columns: [...], index: [...], data: [[...]]}` | 保留完整元信息          |
| `'table'`   | JSON Schema + data                          | 带 schema 的跨语言交换      |

### 综合示例

#### 示例代码

```python
import pandas as pd
import os

df = pd.DataFrame({
    "Name": ["Alice", "Bob"],
    "Age": [25, 30],
    "Skills": [["Python", "SQL"], ["Java", "C++"]],
})

df.to_json("temp_demo.json", orient="records", indent=2, force_ascii=False)
df_read = pd.read_json("temp_demo.json")
print(df_read)

os.remove("temp_demo.json")
```

#### 输出

```text
    Name  Age         Skills
0  Alice   25  [Python, SQL]
1    Bob   30    [Java, C++]
```

#### 理解重点

- 含中文时**必须** `force_ascii=False`，否则会写成 `\u4e2d\u6587` 形式。
- `orient='records'` 是与 Web API 对接时最自然的格式。
- 嵌套列表 / 字典列可以被序列化，但读回时变成 `object` dtype。

## 其他导出格式

### `df.to_html`

#### 作用

导出为 HTML 表格，便于嵌入网页或邮件报告。

#### 重点方法

```python
df.to_html(buf=None, columns=None, col_space=None, header=True,
           index=True, na_rep='NaN', float_format=None, bold_rows=True,
           classes=None, escape=True, notebook=False, border=None,
           table_id=None, render_links=False, index_names=True)
```

### `df.to_sql`

#### 作用

将 DataFrame 写入 SQL 数据库。需要 SQLAlchemy `Engine` 或连接对象。

#### 重点方法

```python
df.to_sql(name, con, schema=None, if_exists='fail', index=True,
          index_label=None, chunksize=None, dtype=None, method=None)
```

#### 参数

| 参数名      | 本例取值                         | 说明                                                        |
| ----------- | -------------------------------- | ----------------------------------------------------------- |
| `name`      | `"table_name"`                   | 表名                                                        |
| `con`       | SQLAlchemy `Engine` / connection | 数据库连接                                                  |
| `schema`    | `None`（默认）                   | 数据库 schema                                               |
| `if_exists` | `'fail'`（默认）、`'replace'`、`'append'` | 表已存在时的行为                                   |
| `index`     | `True`（默认）、`False`          | 是否写入行索引                                              |
| `chunksize` | `None`（默认）、`1000`           | 分批写入，每批行数                                          |
| `method`    | `None`、`'multi'`                | 插入方法；`'multi'` 用单条多行 `INSERT`，通常更快           |

### `df.to_pickle` / `pd.read_pickle`

#### 作用

用 Python 的 `pickle` 序列化。**读写最快**但仅适合 Python 内部使用，跨版本 / 跨语言都有风险。

#### 重点方法

```python
df.to_pickle(path, compression='infer', protocol=5, storage_options=None)
pd.read_pickle(filepath_or_buffer, compression='infer', storage_options=None)
```

### `df.to_parquet`

#### 作用

列存二进制格式，**大数据场景的首选**。列式压缩比高、按列读取快、保留完整 dtype。

#### 重点方法

```python
df.to_parquet(path=None, engine='auto', compression='snappy', index=None,
              partition_cols=None, storage_options=None)
```

#### 参数

| 参数名           | 本例取值                  | 说明                                           |
| ---------------- | ------------------------- | ---------------------------------------------- |
| `path`           | `"data.parquet"`          | 文件路径                                       |
| `engine`         | `'auto'`（默认）、`'pyarrow'`、`'fastparquet'` | 写入引擎                   |
| `compression`    | `'snappy'`（默认）、`'gzip'`、`'brotli'`、`None` | 压缩算法               |
| `index`          | `None`（默认）            | 是否写入索引；`False` 时不写                   |
| `partition_cols` | `None`、`['year', 'month']` | 按列分区存储（生成目录结构）               |

### `df.to_string`

#### 作用

转为格式化字符串，适合日志、控制台输出。可精确控制列宽、对齐、缺失值显示。

#### 重点方法

```python
df.to_string(buf=None, columns=None, col_space=None, header=True,
             index=True, na_rep='NaN', formatters=None, float_format=None,
             sparsify=None, index_names=True, justify=None, max_rows=None,
             max_cols=None, show_dimensions=False, line_width=None)
```

### 格式选择建议

| 场景                     | 推荐格式        | 理由                           |
| ------------------------ | --------------- | ------------------------------ |
| Excel / 手工编辑         | CSV / Excel     | 通用、可读                     |
| Web API 交互             | JSON            | 与 JS 原生兼容                 |
| Python 内部持久化（短期） | Pickle          | 最快、保留所有类型             |
| 大数据存储 / 读写        | Parquet         | 列存高效、压缩好               |
| 数据仓库                 | SQL             | 配合数据库查询                 |
| 报告嵌入                 | HTML            | 直接呈现                       |
| 日志 / 终端输出          | `to_string()`   | 可控格式                       |

## 常见坑

1. `to_csv` 默认 `index=True`，再次读回时会多出 `Unnamed: 0` 列；**绝大多数场景应设 `index=False`**。
2. `read_csv` 中文乱码：依次尝试 `'utf-8'`、`'utf-8-sig'`、`'gbk'`、`'gb18030'`。
3. `read_excel` 缺少引擎会抛 `ImportError`：`.xlsx` 装 `openpyxl`，旧版 `.xls` 装 `xlrd`。
4. JSON 中文乱码：`to_json` 忘记 `force_ascii=False` 会输出 `\uXXXX` 转义序列。
5. `read_json` 的 `orient` 必须与写入时一致，否则解析错位。
6. `to_sql` 默认 `if_exists='fail'`，表已存在会报错；想覆盖用 `'replace'`，想追加用 `'append'`。
7. Parquet 需要额外装 `pyarrow` 或 `fastparquet`。

## 小结

- IO 是数据分析的起点和终点，Pandas 提供覆盖面极广的读写接口。
- CSV / JSON 通用但文本；Pickle / Parquet 二进制但更高效。
- 处理大文件时优先考虑 `chunksize` 分块和 `usecols` 列选择。
- 写入端明确格式参数，读取端用相同参数回读——是避免隐式错误的关键。
